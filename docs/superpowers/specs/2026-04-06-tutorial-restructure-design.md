# Claude Code 中文教程重构设计方案

> **创建日期**：2026-04-06
> **状态**：待审批

---

## 一、项目背景

### 1.1 当前问题

Claude Code 中文教程目前包含 18 个模块，内容丰富但存在以下问题：

| 问题类型 | 具体表现 |
|----------|----------|
| **缺少入门引导** | 第一章就是 CLI 参考手册，新读者不知道从哪里开始 |
| **内容风格混乱** | 多数章节是"参考手册"风格（命令表格罗列），而非"教程"风格（场景驱动） |
| **缺少叙事逻辑** | 章节之间没有"为什么下一章学这个"的逻辑关联 |
| **读者定位模糊** | 没有明确目标读者是谁、学到什么程度 |

### 1.2 重构目标

将教程从"功能参考手册"转变为"系统化学习教程"：

- **目标读者**：想要全面掌握 Claude Code 的人
- **学习目标**：精通并扩展（自定义 Agent、写 Plugin、搭建企业级工作流）
- **叙事方式**：问题驱动式（每章解决一个问题，引出下一章）
- **内容风格**：场景 + 实践结合（每场景后有 Try It Now）

---

## 二、设计原则

### 2.1 教程设计原则

1. **读者导向**：从读者需求出发，而非功能列表
2. **问题驱动**：每章解决一个明确的问题
3. **循序渐进**：从简单到复杂，建立学习路径
4. **场景切入**：用真实场景引入概念，而非抽象定义
5. **即时实践**：每个概念配合动手练习

### 2.2 章节结构规范

每章遵循统一结构：

```markdown
# [章节名称]

> 难度等级 | 预计时间

**你将学会：** [一句话目标]

## 为什么需要这个？
[解决什么问题，为什么重要]

## 核心概念
[简明解释]

## 实战场景
### 场景 1：[名称]
### 场景 2：[名称]
...

## 🎯 Try It Now
[动手练习]

## 常见问题

## 下一章预告
[引出下一章]
```

### 阶段一：上手

#### 第 01 章：快速开始

| 项目 | 内容 |
|------|------|
| **解决的问题** | "Claude Code 是什么？能帮我做什么？怎么开始用？" |
| **读者获得** | 5 分钟体验 Claude Code 的核心价值，知道它能不能帮到自己 |
| **难度** | 🟢 初级 |
| **预计时间** | 30 分钟 |

**核心场景**：
- 场景1：第一次对话 — 让 Claude 解释一段代码
- 场景2：让 Claude 写代码 — 实现一个简单功能
- 场景3：让 Claude 改代码 — 修复一个 Bug

**内容大纲**：
1. Claude Code 是什么
   - 不是 ChatGPT，是 AI 编程伙伴
   - 能读代码、写代码、改代码、运行命令
   
2. 安装（5分钟）
   - macOS / Windows / Linux 三种方式
   
3. 第一次对话
   - 场景：让 Claude 解释一段代码
   - Try It Now
   
4. 能帮你做什么
   - 代码审查、Bug修复、功能开发、文档生成、测试编写...
   
5. 学习路线图
   - 四个阶段的概览

**引出下一章**："我已经会用了，但每次都要输入完整的问题，有没有更高效的方式？"

#### 第 02 章：交互与对话

| 项目 | 内容 |
|------|------|
| **解决的问题** | "怎么和 Claude 高效沟通？怎么管理对话？" |
| **读者获得** | 掌握高效交互方式，会管理会话 |
| **难度** | 🟢 初级 |
| **预计时间** | 45 分钟 |

**核心场景**：
- 场景1：启动会话 — 不同启动方式的区别
- 场景2：继续上次对话 — 会话恢复
- 场景3：用 Print 模式快速提问 — 非交互式使用

**内容大纲**：
1. 交互式 vs Print 模式
   - 各自适用场景
   
2. 会话管理
   - 命名会话：`claude -n "auth-refactor"`
   - 恢复会话：`claude -c` 或 `claude -r <id>`
   - 继续会话：保持上下文连续性
   
3. 高效沟通技巧
   - 怎么提问更清晰
   - 怎么给上下文（引用文件、描述背景）
   - 怎么逐步细化需求
   
4. 常用 CLI 命令（精简版）
   - 只讲最常用的 10 个命令
   - 完整命令列表见附录 A
   
5. Try It Now
   - 练习：创建命名会话，退出后恢复

**引出下一章**："每次都要输入完整指令太麻烦，有没有快捷命令？"
