Hooks 自动化

自动格式化 Hook

{
  "hooks": [
    {
      "id": "auto-format",
      "event": "PostToolUse",
      "conditions": [
        { "type": "tool", "value": "write_file" },
        { "type": "file", "value": "*.{ts,tsx,js,jsx}" }
      ],
      "type": "command",
      "command": "npx prettier --write ${file}",
      "timeout": 10000
    }
  ]
}

自动测试 Hook

{
  "hooks": [
    {
      "id": "auto-test",
      "event": "PostToolUse",
      "conditions": [
        { "type": "tool", "value": "write_file" },
        { "type": "file", "value": "src/**/*.{ts,tsx}" }
      ],
      "type": "function",
      "function": "async (ctx) => { const file = ctx.data.params.path; const testFile = file.replace('/src/', '/tests/').replace('.ts', '.test.ts'); if (await fileExists(testFile)) { await runCommand(`npm test -- ${testFile}`); } }"
    }
  ]
}

自动生成 Commit 消息

{
  "hooks": [
    {
      "id": "generate-commit-msg",
      "event": "PrePrompt",
      "conditions": [
        { "type": "pattern", "value": "commit|提交" }
      ],
      "type": "function",
      "function": "async (ctx) => { const diff = await runCommand('git diff --cached'); const msg = await generateCommitMessage(diff); return { modify: { ...ctx.data, suggestedMessage: msg } }; }"
    }
  ]
}

Skill 工作流

功能开发 Skill

# skills/feature-development/skill.yaml
id: feature-development
name: Feature Development Workflow
version: 1.0.0
description: 完整的功能开发工作流

triggers:
  - type: command
    value: /feature

execution:
  mode: interactive
  maxSteps: 100

prompts:
  system: |
    你是一个功能开发助手，帮助开发者完成从设计到实现的完整流程。
    
    工作流程：
    1. 需求分析：理解功能需求，确认边界条件
    2. 技术设计：设计数据结构、API 接口、组件结构
    3. 编码实现：按照设计编写代码
    4. 测试验证：编写单元测试和集成测试
    5. 文档更新：更新相关文档
    
    每个阶段都需要用户确认后再继续。

dependencies:
  tools:
    - read_file
    - write_file
    - search_files
    - shell_exec

CI/CD 集成

GitHub Actions 集成

# .github/workflows/claude-review.yml
name: Claude Code Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Run Code Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          claude --non-interactive \
            "Review the changes in this PR. Focus on:
            1. Code quality and best practices
            2. Potential bugs or issues
            3. Security concerns
            4. Performance implications
            
            Provide a summary and specific suggestions." \
            > review_output.md

      - name: Post Review Comment
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const review = fs.readFileSync('review_output.md', 'utf8');
            await github.rest.issues.createComment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
              body: `## 🤖 Claude Code Review\n\n${review}`
            });

自动测试报告

# .github/workflows/test-report.yml
name: Test Report

on:
  push:
    branches: [main, develop]
  pull_request:

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Run Tests
        run: npm test -- --json --outputFile=test-results.json

      - name: Generate Report
        if: always()
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          claude --non-interactive \
            "Analyze the test results in test-results.json and provide:
            1. Summary of passed/failed tests
            2. Analysis of failures
            3. Suggestions for fixing issues
            4. Recommendations for improving test coverage" \
            > test-report.md

      - name: Upload Report
        uses: actions/upload-artifact@v4
        with:
          name: test-report
          path: test-report.md

自动化脚本库

项目初始化脚本

#!/bin/bash
# scripts/init-project.sh

set -e

echo "🚀 Initializing project with Claude Code..."

# 创建基础目录结构
mkdir -p src/{components,hooks,services,utils,types,tests}
mkdir -p docs

# 创建 CLAUDE.md
cat > CLAUDE.md << 'EOF'
# 项目名称

## 技术栈
- React + TypeScript
- Tailwind CSS
- Jest + React Testing Library

## 目录结构
src/
├── components/  # React 组件
├── hooks/       # 自定义 Hooks
├── services/    # API 服务
├── utils/       # 工具函数
├── types/       # 类型定义
└── tests/       # 测试文件
EOF

# 创建 .claudeignore
cat > .claudeignore << 'EOF'
node_modules/
dist/
.env*
*.log
EOF

# 初始化 Git
git init
git add .
git commit -m "chore: initial commit"

echo "✅ Project initialized successfully!"

代码生成脚本

#!/bin/bash
# scripts/generate-component.sh

COMPONENT_NAME=$1

if [ -z "$COMPONENT_NAME" ]; then
    echo "Usage: ./generate-component.sh ComponentName"
    exit 1
fi

# 使用 Claude Code 生成组件
claude --non-interactive \
    "Create a React component named ${COMPONENT_NAME} with:
    - TypeScript interface for props
    - Basic styling with Tailwind CSS
    - Unit test file
    - Storybook story
    
    Follow the project's coding standards in CLAUDE.md." \
    --output src/components/${COMPONENT_NAME}.tsx

echo "✅ Component ${COMPONENT_NAME} created!"

自动化最佳实践

渐进式自动化

  1. 从简单的自动化开始（格式化、lint）
  2. 逐步添加更复杂的自动化（测试、审查）
  3. 验证自动化结果后再扩大范围
  4. 保留人工确认的关键节点

错误处理

  • 自动化失败时提供清晰的错误信息
  • 实现回退机制避免阻塞开发
  • 记录自动化日志便于排查问题
  • 设置合理的超时时间

性能考虑

  • 避免过度自动化影响响应速度
  • 使用增量检查减少重复工作
  • 并行执行独立的自动化任务
  • 缓存中间结果
