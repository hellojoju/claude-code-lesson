开发环境配置

可靠性: A 来源: Claude Code 官方文档 + 社区最佳实践

配置目标

一个优秀的开发环境配置可以让 Claude Code 更好地理解项目上下文，提供更精准的代码建议。本节将手把手教你配置高效的开发环境。

第一步：项目级配置

1.1 创建 CLAUDE.md

CLAUDE.md 是 Claude Code 理解项目的关键文件，应该包含项目的核心信息：

# 项目名称

## 项目概述
简要描述项目的目的和主要功能。

## 技术栈
- 前端: React 18 + TypeScript + Tailwind CSS
- 后端: Node.js + Express + PostgreSQL
- 测试: Jest + React Testing Library
- 部署: Docker + Kubernetes

## 目录结构
```
src/
├── components/     # React 组件
│   ├── common/     # 通用组件
│   └── features/   # 功能组件
├── hooks/          # 自定义 Hooks
├── services/       # API 服务
├── utils/          # 工具函数
├── types/          # TypeScript 类型定义
└── tests/          # 测试文件
```

## 编码规范
- 使用函数式组件和 Hooks
- 遵循 ESLint 和 Prettier 配置
- 组件命名使用 PascalCase
- 函数命名使用 camelCase
- 常量使用 UPPER_SNAKE_CASE

## API 规范
- RESTful API 设计
- 统一响应格式: `{ code, data, message }`
- 错误码规范: 1000-1999 业务错误, 2000-2999 系统错误

## 数据库规范
- 表名使用 snake_case
- 主键使用 `id`
- 时间字段: `created_at`, `updated_at`
- 软删除字段: `deleted_at`

## Git 规范
- 分支命名: `feature/xxx`, `fix/xxx`, `refactor/xxx`
- Commit 格式: `type(scope): description`
- 类型: feat, fix, docs, style, refactor, test, chore

## 重要文件
- `src/index.tsx` - 应用入口
- `src/App.tsx` - 根组件
- `src/api/client.ts` - API 客户端
- `src/utils/constants.ts` - 常量定义

## 已知问题
- [ ] 性能优化: 大列表渲染需要虚拟化
- [ ] 重构: 认证模块需要解耦
- [ ] 文档: API 文档需要更新

## 开发注意事项
1. 修改 API 时需要同步更新类型定义
2. 新组件需要添加 Storybook 故事
3. 重要功能需要编写单元测试
4. 遵循最小权限原则配置 API

2.2 全局 CLAUDE.md

# 全局开发偏好

## 编码风格
- 优先使用 TypeScript
- 函数式编程风格
- 明确的类型定义，避免 any
- 详细的注释和文档

## 代码质量
- 遵循 SOLID 原则
- DRY (Don't Repeat Yourself)
- KISS (Keep It Simple, Stupid)
- 编写可测试的代码

## 安全意识
- 永远不要硬编码敏感信息
- 验证所有用户输入
- 使用参数化查询防止 SQL 注入
- 实现适当的错误处理

## 性能考虑
- 避免不必要的渲染
- 使用缓存优化重复计算
- 懒加载大型组件
- 优化数据库查询

## 测试策略
- 单元测试覆盖核心逻辑
- 集成测试覆盖关键流程
- E2E 测试覆盖用户场景
- 保持测试独立和可重复

第四步：Git 配置

4.1 Git Hooks 配置

# .git/hooks/pre-commit
#!/bin/bash

# 运行 lint 检查
npm run lint

# 运行类型检查
npm run typecheck

# 运行测试
npm run test -- --bail --findRelatedTests

# 如果失败，阻止提交
if [ $? -ne 0 ]; then
    echo "Pre-commit checks failed. Please fix the issues before committing."
    exit 1
fi

4.2 Commit 模板

# .gitmessage
# type(scope): subject
#
# body
#
# footer
#
# Type:
#   feat     - 新功能
#   fix      - 修复 bug
#   docs     - 文档变更
#   style    - 代码格式
#   refactor - 重构
#   test     - 测试
#   chore    - 构建/工具
#
# Scope: 组件/模块名称
#
# Subject: 简短描述（50字符以内）
#
# Body: 详细描述（可选）
#
# Footer: 关联 issue（可选）

第五步：自动化脚本

5.1 package.json 脚本

{
  "scripts": {
    "dev": "vite",
    "build": "tsc && vite build",
    "preview": "vite preview",
    "lint": "eslint src --ext .ts,.tsx",
    "lint:fix": "eslint src --ext .ts,.tsx --fix",
    "typecheck": "tsc --noEmit",
    "test": "jest",
    "test:watch": "jest --watch",
    "test:coverage": "jest --coverage",
    "format": "prettier --write \"src/**/*.{ts,tsx,css}\"",
    "prepare": "husky install",
    "precommit": "lint-staged"
  },
  "lint-staged": {
    "*.{ts,tsx}": [
      "eslint --fix",
      "prettier --write"
    ]
  }
}
