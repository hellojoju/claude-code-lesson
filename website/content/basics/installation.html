安装与配置

可靠性: A 来源: Claude Code 官方文档

预计阅读时间: 30 分钟 | 难度: 初级

系统要求检查

在安装 Claude Code 之前，请确保您的系统满足以下要求：

macOS 用户

打开终端，运行以下命令检查系统版本：

sw_vers

预期输出：

ProductName:    macOS
ProductVersion: 14.5.1  # 需要版本 >= 10.15
BuildVersion:   23F79

如果您的版本低于 10.15，请先升级 macOS。

Ubuntu/Debian 用户

运行以下命令检查系统版本：

lsb_release -a

预期输出：

No LSB modules are available.
Distributor ID: Ubuntu
Description:    Ubuntu 22.04.4 LTS  # 需要版本 >= 20.04
Release:        22.04
Codename:       jammy

Windows 用户

运行以下命令检查 WSL 安装状态：

wsl --version

预期输出：

WSL 版本： 2.2.4.0
内核版本： 5.15.146.1-2
WSLg 版本： 1.0.60
MSRDC 版本： 1.2.5105
Direct3D 版本： 1.611.1-81528511
DXCore 版本： 10.0.25131.1002-220531-1250
Windows 版本： 10.0.22631.3296

如果未安装 WSL，请先安装：wsl --install

Node.js 检查

Claude Code 需要 Node.js 18+，运行以下命令检查：

node --version

预期输出：

v20.12.2  # 需要版本 >= 18.0.0

如果未安装或版本过低，请访问 nodejs.org 下载安装。

npm 检查

运行以下命令检查 npm 版本：

npm --version

预期输出：

10.5.0

安装方法

方法一：Claude Desktop App（推荐新手）

步骤 1：下载安装包

打开浏览器，访问官方下载页面：

https://claude.ai/download

点击对应操作系统的下载按钮。

步骤 2：运行安装程序

macOS:
  1. 双击下载的 .dmg 文件
  2. 将 Claude 图标拖到 Applications 文件夹
  3. 打开 Applications 文件夹，双击 Claude
Windows:
  1. 双击下载的 .exe 文件
  2. 按照安装向导完成安装
  3. 安装完成后，Claude 会自动启动
Linux:
  1. 运行下载的 .deb 或 .rpm 文件
  2. 或使用命令：sudo dpkg -i claude-desktop.deb

步骤 3：登录账号

安装完成后，打开 Claude Desktop App：

  1. 如果您已有 Anthropic 账号，点击"Sign In"登录
  2. 如果没有账号，点击"Sign Up"注册新账号
  3. 按照提示完成邮箱验证

步骤 4：验证安装成功

打开终端，运行以下命令验证：

claude --version

预期输出：

claude-code version 2.1.88

恭喜！您已成功安装 Claude Code！

方法二：命令行安装（推荐开发者）

步骤 1：检查 Node.js 版本

node --version

预期输出：

v18.x.x 或更高

如果版本低于 18，请先升级 Node.js。

步骤 2：全局安装 Claude Code

npm install -g @anthropic-ai/claude-code

预期输出：

added 1 package in 5s

1 package is looking for funding
  run `npm fund` for details

如果遇到权限错误，请使用：sudo npm install -g @anthropic-ai/claude-code

步骤 3：验证安装

claude --version

预期输出：

claude-code version x.x.x

方法三：源码安装（高级用户）

步骤 1：克隆仓库

git clone https://github.com/anthropics/claude-code.git
cd claude-code

步骤 2：安装依赖

npm install

步骤 3：构建项目

npm run build

步骤 4：全局链接

npm link

国内使用方法

重要：国内用户需要特殊配置才能正常使用 Claude Code。

方法一：使用 Claude Code Router（推荐）

步骤 1：安装 Claude Code Router

npm install -g claude-code-router

步骤 2：初始化配置

ccr init

选择选项 1 进行初始化。

步骤 3：打开 Web 界面

ccr web

浏览器会自动打开 http://localhost:8080

步骤 4：添加供应商

  1. 在 Web 界面中点击"添加供应商"
  2. 选择"从模板导入"
  3. 选择你的 API 供应商（如 OpenAI、Azure 等）
  4. 输入 API 密钥和代理地址
  5. 保存配置

步骤 5：重启 Claude Code

claude

现在可以正常使用 Claude Code 了！

方法二：配置代理服务器

步骤 1：设置代理环境变量

export https_proxy=http://127.0.0.1:7890
export http_proxy=http://127.0.0.1:7890

请将 127.0.0.1:7890 替换为您的代理地址。

步骤 2：设置 API 密钥

export ANTHROPIC_API_KEY=sk-ant-xxxxx

API 密钥需要以 sk-ant- 开头。

步骤 3：验证连接

claude -p "hello"

预期输出：

Hello! How can I help you today?

方法三：使用国内镜像

步骤 1：配置 npm 镜像源

npm config set registry https://registry.npmmirror.com

步骤 2：修改 API 请求地址

在项目根目录创建 .env 文件：

ANTHROPIC_BASE_URL=https://your-mirror-url.com
ANTHROPIC_API_KEY=sk-ant-xxxxx

环境变量配置

ANTHROPIC_API_KEY

说明：Anthropic API 密钥，必需。

获取方法：

  1. 访问 console.anthropic.com
  2. 登录您的账号
  3. 在 API Keys 页面创建新密钥

配置方法：

# macOS/Linux
export ANTHROPIC_API_KEY=sk-ant-xxxxx

# Windows (PowerShell)
$env:ANTHROPIC_API_KEY="sk-ant-xxxxx"

# Windows (CMD)
set ANTHROPIC_API_KEY=sk-ant-xxxxx

ANTHROPIC_BASE_URL

说明：API 请求地址，可选（用于代理）。

配置方法：

export ANTHROPIC_BASE_URL=https://your-proxy-url.com

其他可选环境变量

# 代理设置
export https_proxy=http://127.0.0.1:7890
export http_proxy=http://127.0.0.1:7890

# 模型选择
export ANTHROPIC_MODEL=claude-3-5-sonnet-20241022

# 最大令牌数
export ANTHROPIC_MAX_TOKENS=4096

首次启动和初始化

步骤 1：进入项目目录

cd /path/to/your/project

步骤 2：启动 Claude Code

claude

预期输出：

Welcome to Claude Code!
Version: 2.1.88

Do you trust this project? (yes/no)

步骤 3：确认信任项目

yes

Claude Code 会扫描项目文件，这可能需要几秒钟。

步骤 4：开始使用

看到提示符 > 表示启动成功，您可以开始输入指令了：

> 帮我分析这个项目的结构

常见问题排查

问题 1：npm ERR! permission denied

原因：权限不足。

解决方案：

# 方法1：使用 sudo
sudo npm install -g @anthropic-ai/claude-code

# 方法2：修改 npm 全局目录
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

问题 2：Error: Invalid API Key

原因：API 密钥格式错误或未设置。

解决方案：

  1. 检查密钥格式，确保以 sk-ant- 开头
  2. 确认密钥没有多余的空格或换行符
  3. 验证环境变量是否正确设置： 
    echo $ANTHROPIC_API_KEY

问题 3：Network request failed

原因：网络连接问题。

解决方案：

  1. 检查网络连接
  2. 配置代理： 
    export https_proxy=http://127.0.0.1:7890
  3. 检查防火墙设置
  4. 尝试使用国内镜像

问题 4：Permission denied

原因：.claude 目录权限问题。

解决方案：

# 修改权限
chmod -R 755 ~/.claude

# 或删除重建
rm -rf ~/.claude
claude  # 重新初始化

问题 5：Node.js version mismatch

原因：Node.js 版本过低。

解决方案：

# 使用 nvm 升级 Node.js
nvm install 20
nvm use 20

# 或使用 n
n 20
