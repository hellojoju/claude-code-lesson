权限管理详解

来源说明：本文内容基于 Claude Code 官方文档和安全最佳实践整理。

可靠性评级：A 官方文档 + 安全专家审核

权限管理概述

Claude Code 作为系统级 AI Agent，具有执行代码、读写文件、运行系统命令的能力。权限管理是确保安全使用 Claude Code 的关键环节。

为什么需要权限管理？

  • 数据安全：防止意外或恶意修改敏感文件
  • 系统安全：限制危险命令的执行
  • 合规要求：满足企业安全审计需求
  • 可控性：精确控制 AI 的操作范围

权限模型

权限层级

Claude Code 采用分层权限模型：

项目级权限 (最高优先级)
    ↓
用户级权限
    ↓
系统级权限 (最低优先级)

权限类型

权限类型 说明 风险等级
file:read 读取文件内容 低
file:write 写入/修改文件 中
file:delete 删除文件 高
command:execute 执行系统命令 高
network:access 网络访问 中

权限配置文件

配置文件位置

~/.claude/permissions.json    # 用户级权限
.project/permissions.json     # 项目级权限

基础配置示例

{
  "version": "1.0",
  "defaultMode": "ask",
  "permissions": {
    "file:read": {
      "mode": "allow",
      "paths": ["./**"]
    },
    "file:write": {
      "mode": "ask",
      "paths": ["./src/**", "./tests/**"],
      "exclude": ["./src/config/secrets.json"]
    },
    "file:delete": {
      "mode": "deny"
    },
    "command:execute": {
      "mode": "ask",
      "allowedCommands": ["npm", "git", "node"],
      "deniedCommands": ["rm -rf", "sudo", "chmod"]
    }
  }
}

权限模式详解

三种权限模式

allow - 允许模式

自动允许操作，无需确认。适用于低风险操作。

"file:read": {
  "mode": "allow",
  "paths": ["./**"]
}

ask - 询问模式（推荐）

每次操作前询问用户确认。平衡安全与效率。

"file:write": {
  "mode": "ask",
  "paths": ["./src/**"]
}

deny - 拒绝模式

完全禁止操作。适用于高风险操作。

"file:delete": {
  "mode": "deny"
}

路径规则配置

路径匹配语法

./**          # 当前目录及所有子目录
./src/**      # src 目录及所有子目录
*.js          # 所有 .js 文件
./config/*    # config 目录下的直接文件

包含与排除规则

{
  "file:write": {
    "mode": "ask",
    "paths": ["./**"],
    "exclude": [
      "./.env",
      "./secrets/**",
      "./**/credentials.json",
      "./**/*.key",
      "./**/*.pem"
    ]
  }
}

敏感文件保护

建议始终排除以下类型的文件：

{
  "exclude": [
    "./.env",
    "./.env.*",
    "./**/secrets/**",
    "./**/credentials/**",
    "./**/*.key",
    "./**/*.pem",
    "./**/*.p12",
    "./**/id_rsa*",
    "./**/authorized_keys"
  ]
}

命令执行控制

允许特定命令

{
  "command:execute": {
    "mode": "ask",
    "allowedCommands": [
      "npm",
      "npm run *",
      "npm test",
      "git status",
      "git diff",
      "git log",
      "node",
      "npx"
    ]
  }
}

禁止危险命令

{
  "command:execute": {
    "mode": "ask",
    "deniedCommands": [
      "rm -rf",
      "rm -rf /",
      "sudo *",
      "chmod 777",
      "chown",
      "dd *",
      "mkfs *",
      "> /dev/sda",
      "curl * | bash",
      "wget * | bash"
    ]
  }
}

命令模式组合

{
  "command:execute": {
    "mode": "ask",
    "rules": [
      {
        "pattern": "npm *",
        "mode": "allow",
        "description": "允许所有 npm 命令"
      },
      {
        "pattern": "git status",
        "mode": "allow",
        "description": "允许查看 git 状态"
      },
      {
        "pattern": "git push *",
        "mode": "ask",
        "description": "推送前需要确认"
      },
      {
        "pattern": "rm *",
        "mode": "deny",
        "description": "禁止删除命令"
      }
    ]
  }
}

交互式权限确认

确认提示示例

当 Claude Code 需要执行需要确认的操作时，会显示：

┌─────────────────────────────────────────┐
│ Claude 想要执行以下操作：                  │
│                                         │
│ 操作类型: 文件写入                        │
│ 目标文件: ./src/index.js                 │
│ 风险等级: 中                             │
│                                         │
│ 是否允许？                               │
│ [y] 是    [n] 否    [a] 总是允许         │
│ [d] 查看详情    [e] 编辑后执行            │
└─────────────────────────────────────────┘

快捷键说明

按键 说明
y 允许本次操作
n 拒绝本次操作
a 总是允许此类操作（添加到白名单）
d 查看操作详情
e 编辑后执行（适用于文件写入）

会话权限模式

自动接受模式

# 启动时使用自动接受模式（谨慎使用）
claude --auto-accept

# 或在会话中切换
> /mode auto-accept

警告：自动接受模式会跳过所有权限确认，仅建议在完全信任的环境中使用。

严格模式

# 启动时使用严格模式
claude --strict

# 或在会话中切换
> /mode strict

交互模式

# 默认模式，每次操作询问
claude

# 或在会话中切换
> /mode interactive

权限审计日志

启用审计日志

{
  "audit": {
    "enabled": true,
    "logFile": "~/.claude/audit.log",
    "level": "verbose",
    "include": [
      "file:write",
      "file:delete",
      "command:execute"
    ]
  }
}

审计日志格式

2026-04-04T10:30:15Z [INFO] file:write ./src/index.js ALLOWED
2026-04-04T10:30:45Z [WARN] command:execute "rm -rf ./dist" DENIED
2026-04-04T10:31:00Z [INFO] file:read ./package.json ALLOWED

查看审计日志

# 查看最近的权限操作
tail -f ~/.claude/audit.log

# 搜索被拒绝的操作
grep "DENIED" ~/.claude/audit.log

# 搜索特定类型的操作
grep "file:delete" ~/.claude/audit.log

常见问题

权限配置不生效

解决方案：

  1. 检查配置文件路径是否正确
  2. 验证 JSON 语法是否有效
  3. 重启 Claude Code 会话
  4. 使用 /config reload 重新加载配置

误拦截安全操作

解决方案：

  1. 检查排除规则是否过于严格
  2. 添加特定路径到允许列表
  3. 使用 a 键添加到白名单
