基本使用

可靠性: A 来源: Claude Code 官方文档

预计阅读时间: 25 分钟 | 难度: 初级

启动 Claude Code

步骤 1：打开终端

在您的项目目录中打开终端。

cd /path/to/your/project

步骤 2：启动 Claude Code

claude

预期输出：

Welcome to Claude Code!
Version: 2.1.88

Do you trust this project? (yes/no)

步骤 3：确认信任项目

yes

Claude Code 会扫描项目文件，这可能需要几秒钟。

步骤 4：开始使用

看到提示符 > 表示启动成功，您可以开始输入指令了。

使用模式

交互模式（推荐）

启动：

claude

特点：

  • 支持多轮对话
  • 可以保存和加载会话
  • 适合复杂任务

继续上次会话

启动：

claude -c

特点：

  • 自动加载上次的会话历史
  • 保持上下文连续性
  • 适合长期项目

单次模式

用法：

claude -p "你的问题或任务"

示例：

claude -p "帮我创建一个 README.md 文件"
claude -p "分析 package.json 中的依赖"

特点：

  • 快速执行单个任务
  • 不需要进入交互模式
  • 适合简单查询

快速模式（跳过权限确认）

启动：

claude --dangerously-skip-permissions
警告：此模式会自动执行所有操作，请谨慎使用！建议仅在受信任的环境中使用。

基本操作

文件操作

读取文件：

> /read package.json

写入文件：

> /write hello.txt "Hello, World!"

编辑文件：

> /edit config.json

列出目录：

> /ls

代码执行

运行代码：

> /run

执行系统命令：

> /exec "npm install"
> /exec "git status"

会话管理

保存会话：

> /save my-project-session

加载会话：

> /load my-project-session

清除会话：

> /clear

自然语言指令

Claude Code 支持自然语言指令，您可以用日常语言描述任务：

示例 1：创建功能

> 我想添加一个用户登录功能，包括：
> 1. 登录表单
> 2. 验证逻辑
> 3. 错误处理

示例 2：修复 Bug

> 我遇到了一个错误：TypeError: Cannot read property 'x' of undefined
> 请帮我找到问题并修复

示例 3：代码重构

> 请重构 src/utils.js 文件，使其更易读和维护

示例 4：代码解释

> 请解释这段代码的作用：
> [粘贴代码]

最佳实践

1. 清晰描述需求

越具体的描述，得到的结果越好：

# ❌ 不好的例子
> 帮我写个函数

# ✅ 好的例子
> 创建一个验证邮箱格式的函数，要求：
> - 接受字符串参数
> - 返回布尔值
> - 支持常见邮箱格式

2. 使用 Plan 模式

对于复杂任务，先规划再执行：

> /plan
> 我想创建一个完整的用户管理系统

3. 分步骤执行

将大任务拆分成小步骤：

> 步骤1：创建数据库模型
> 步骤2：实现 API 接口
> 步骤3：添加前端页面
> 步骤4：编写测试用例

4. 及时保存会话

定期保存会话，避免丢失工作进度：

> /save project-2024-01-15

实战示例

示例 1：创建 Node.js 项目

> 我想创建一个简单的 Node.js 项目，包括：
> 1. package.json 文件
> 2. index.js 主文件
> 3. README.md 说明文档

# Claude Code 会：
# - 创建 package.json 并初始化
# - 创建 index.js 并添加基本代码
# - 创建 README.md 并添加项目说明

示例 2：添加测试

> 为 src/calculator.js 文件添加单元测试

# Claude Code 会：
# - 分析 calculator.js 的功能
# - 创建测试文件
# - 编写测试用例
# - 运行测试验证

示例 3：代码优化

> 优化 src/api.js 的性能，目前响应时间太长

# Claude Code 会：
# - 分析代码瓶颈
# - 提出优化建议
# - 实施优化方案
# - 测试性能改进
