会话管理详解

可靠性: A 来源: Claude Code 官方文档

更新时间: 2025-04-04 作者: Claude Code 教程团队

概述

Claude Code 会自动保存所有会话历史，让你可以随时恢复之前的工作状态。会话管理是高效使用 Claude Code 的关键技能之一。

核心概念：会话（Session）是你与 Claude 的完整对话记录，包括所有消息、文件操作和命令执行。

保存会话

Claude Code 会自动保存你的会话，无需手动操作。

自动保存机制

  • 每条消息都会自动保存
  • 会话存储在 ~/.claude/projects/ 目录
  • 支持跨多个项目的独立会话

会话存储位置

# 会话存储结构
~/.claude/
├── projects/
│   ├── project-a/
│   │   ├── session-1.jsonl
│   │   └── session-2.jsonl
│   └── project-b/
│       └── session-1.jsonl
└── settings.json

加载会话

你可以随时恢复之前的会话，继续之前的工作。

继续上次会话

# 方式一：使用 -c 参数
claude -c

# 方式二：使用 --continue
claude --continue

# 方式三：使用 --resume 指定会话
claude --resume session-id

会话列表

# 查看所有会话
> /history

# 或使用命令行
claude --list-sessions

会话选择界面

运行 claude -c 时，会显示最近的会话列表：

Recent sessions:
1. 2025-04-04 10:30 - 添加用户认证功能 (project-a)
2. 2025-04-03 15:20 - 修复登录 bug (project-a)
3. 2025-04-02 09:00 - 重构 API 层 (project-b)

Select a session to continue (1-3):

清除会话

当上下文变得混乱或想开始新任务时，可以清除当前会话。

清除命令

# 清除当前会话内容
> /clear

# 这会：
# - 清空对话历史
# - 保留会话连接
# - 不影响已保存的文件

注意：清除会话后，Claude 会丢失之前的上下文，包括已读取的文件内容和讨论的决策。

何时清除

切换任务

从一个任务切换到完全不同的任务时。

调试问题

上下文变得混乱，需要重新开始时。

上下文过满

达到上下文窗口限制时。

压缩上下文

当会话过长时，可以压缩上下文以节省 token。

压缩命令

# 压缩当前上下文
> /compact

# 带自定义指令压缩
> /compact 保留架构决策，省略调试细节

压缩效果

压缩前 压缩后 节省
100,000 tokens 20,000 tokens 80%
50,000 tokens 10,000 tokens 80%

自动压缩

当上下文接近限制时，Claude Code 会自动提示压缩：

⚠️ 上下文使用率达到 85%，建议使用 /compact 压缩

监控上下文

# 查看当前上下文使用情况
> /context

# 输出示例：
# 上下文使用: 75,000 / 200,000 tokens (37.5%)
# 剩余空间: 125,000 tokens

会话历史

Claude Code 保存完整的会话历史，方便回顾和恢复。

查看历史

# 查看当前会话历史
> /history

# 查看特定会话
claude --view-session session-id

历史记录格式

会话以 JSONL 格式存储，每行一条消息：

{"role": "user", "content": "创建一个登录组件"}
{"role": "assistant", "content": "我来创建登录组件..."}
{"role": "tool", "name": "Write", "input": {...}}

导出会话

# 导出会话为 Markdown
claude --export-session session-id > session.md

# 导出为 JSON
claude --export-session session-id --format json > session.json

最佳实践

按项目分目录

在不同项目目录启动 Claude，会话会自动分开存储。

定期清理

任务完成后使用 /clear 清理，避免上下文污染。

主动压缩

在自然断点（功能完成、提交后）主动压缩上下文。

重要会话

重要决策的会话可以导出保存为文档。

会话管理工作流

# 1. 开始新任务
cd /path/to/project
claude

# 2. 完成任务后
> /compact 保留关键实现细节
> 提交这些更改

# 3. 切换到新任务
> /clear
> 开始下一个任务...

# 4. 恢复之前的工作
claude -c
# 选择要恢复的会话

常见问题

会话太多怎么办？

可以定期清理旧会话：

# 删除 30 天前的会话
find ~/.claude/projects -name "*.jsonl" -mtime +30 -delete

会话丢失了？

检查会话存储目录，确保没有意外删除。会话文件以 .jsonl 格式存储，可以手动恢复。
