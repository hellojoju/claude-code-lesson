可靠性: A 来源: Claude Code 官方文档

更新时间: 2025-04-04 作者: Claude Code 教程团队

概述

Claude Code 可以在你的系统上执行命令和代码，这是它区别于普通 AI 助手的关键能力之一。通过 Bash 工具，Claude 可以运行 shell 命令、执行脚本、管理进程等。

核心能力：执行构建命令、运行测试、启动开发服务器、安装依赖、Git 操作等。

Bash 命令执行

Claude Code 通过 Bash 工具执行 shell 命令，支持所有常见的命令行操作。

基本用法

# 自然语言方式
> 运行 npm install
> 执行 npm test
> 启动开发服务器

常用命令示例

类别 命令示例 说明
包管理 npm install, pip install 安装项目依赖
构建 npm run build, make 构建项目
测试 npm test, pytest 运行测试套件
开发 npm run dev, python app.py 启动开发服务器
Git git status, git commit 版本控制操作

命令权限控制

每次执行命令前，Claude 会请求你的确认。你可以：

  • 允许 - 执行该命令
  • 拒绝 - 取消执行
  • 修改 - 编辑命令后再执行

运行代码

Claude Code 可以直接运行各种编程语言的代码片段。

支持的运行方式

Node.js

> 运行这段 JavaScript 代码
node -e "console.log('Hello, World!')"

Python

> 执行这个 Python 脚本
python3 script.py

Shell

> 运行这个 shell 脚本
bash deploy.sh

交互式命令

对于需要用户输入的交互式命令，Claude Code 会提示你输入：

# 交互式命令示例
> 运行 npm init
# Claude 会提示你输入包名、版本等信息

脚本执行

Claude Code 可以创建并执行脚本文件。

创建并运行脚本

# 让 Claude 创建并运行脚本
> 创建一个部署脚本并执行它
> 写一个数据迁移脚本然后运行

脚本模板

#!/bin/bash
# Claude Code 生成的部署脚本示例

echo "开始部署..."

# 安装依赖
npm install --production

# 构建项目
npm run build

# 运行测试
npm test

# 部署到服务器
rsync -avz dist/ user@server:/var/www/app/

echo "部署完成!"

安全机制

Claude Code 内置了多层安全机制，确保命令执行的安全性。

权限确认

默认行为：所有命令执行前都需要用户确认，防止意外操作。

自动接受模式

按 Shift+Tab 可以切换自动接受模式：

# 自动接受模式
# Claude 会自动执行命令，但仍会显示操作内容

权限配置

在 .claude/settings.json 中配置命令权限：

{
  "permissions": {
    "allowedTools": ["Bash(git:*)", "Bash(npm:*)"],
    "deny": [
      "Bash(rm:*)",
      "Bash(sudo:*)",
      "Bash(git push:*)"
    ]
  }
}

危险命令检测

Claude Code 会自动检测并警告危险命令：

危险命令 风险 建议
rm -rf / 删除系统文件 永远不要执行
sudo ... 提权操作 谨慎使用
git push --force 强制推送 确认后再执行
DROP DATABASE 删除数据库 备份后执行

最佳实践

审查命令

执行前仔细审查 Claude 生成的命令，确保理解其作用。

提交代码

执行可能修改文件的命令前，先提交当前更改。

备份数据

执行数据库操作前，确保有备份。

测试环境

先在测试环境验证，再在生产环境执行。

常见问题

命令执行超时？

对于长时间运行的命令，Claude 会显示进度信息。你可以：

# 使用后台运行
> 在后台启动开发服务器

# 或者使用 nohup
nohup npm run dev &

命令找不到？

确保命令已安装并在 PATH 中。Claude 可以帮你安装：

# 安装缺失的工具
> 安装 ripgrep
brew install ripgrep  # macOS
apt install ripgrep   # Linux
