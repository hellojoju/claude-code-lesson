自然语言指令详解

可靠性: A 来源: Claude Code 官方文档

更新时间: 2025-04-04 作者: Claude Code 教程团队

概述

Claude Code 最强大的特性之一是能够理解自然语言指令。你不需要记忆复杂的命令语法，只需用日常语言描述你想做什么，Claude 就能理解并执行。

核心理念：像和同事交流一样与 Claude 对话，它会理解你的意图并自主完成工作。

基础指令

以下是一些常用的自然语言指令示例：

代码理解

# 理解代码库
> 这个项目是做什么的？
> 主入口在哪里？
> 认证流程是如何工作的？
> 解释 @src/auth/login.ts 的实现逻辑

功能开发

# 开发新功能
> 我想添加一个用户登录功能
> 创建一个带 JWT 认证的 API 端点
> 实现文件上传功能，支持图片和 PDF
> 添加分页功能到用户列表

问题修复

# 修复问题
> 请帮我修复这个错误
> 用户登录时出现 401 错误，请排查
> 这个测试失败了，帮我修复
> 调试这个错误：[粘贴错误信息]

代码重构

# 重构代码
> 将这个组件重构为更模块化的结构
> 把 JavaScript 转换为 TypeScript
> 优化这个函数的性能
> 提取公共逻辑到工具函数

高级指令

更复杂的任务可以通过更详细的描述来实现：

多步骤任务

# 复杂任务分解
> 创建一个完整的用户管理模块：
> 1. 用户注册和登录
> 2. 密码重置功能
> 3. 用户资料编辑
> 4. 权限管理
> 5. 相关的测试用例

带约束的任务

# 带约束的开发
> 创建一个 API 端点，要求：
> - 使用 RESTful 风格
> - 包含输入验证
> - 返回标准化的错误响应
> - 添加日志记录
> - 编写单元测试

思考模式

# 深度思考
> 设计一个微服务架构。ultrathink 关于可扩展性、容错性和性能

> 分析这个安全漏洞。think hard 关于攻击向量和防御策略

思考模式关键词

  • think - 快速架构决策（4000 tokens）
  • think hard / megathink - 多步骤规划（10000 tokens）
  • ultrathink / think harder - 复杂架构决策（31999 tokens）

上下文引用

使用 @ 语法引用文件和目录，让 Claude 理解你的上下文：

引用文件

# 引用单个文件
> 审查 @src/auth/login.ts 的安全问题
> 解释 @package.json 中的依赖关系

# 引用多个文件
> 对比 @src/old.js 和 @src/new.js 的差异
> 分析 @src/api/*.ts 中的 API 端点

引用目录

# 引用目录
> 分析 @src/components/ 的组件结构
> 检查 @tests/ 目录的测试覆盖

引用 MCP 资源

# 引用 MCP 服务器资源
> 查看 @github:repos/owner/repo/issues
> 读取 @notion:page-id 的内容
> 搜索 @slack:channel-name 的消息

提示技巧

具体明确

避免模糊描述，尽量具体说明需求。

# 好
> 在 @src/api/users.ts 添加分页功能

# 差
> 改进用户 API

分步执行

复杂任务分解为多个步骤。

# 好
> 1. 创建数据模型
> 2. 实现 API 端点
> 3. 添加测试

# 差
> 做一个完整的用户系统

提供上下文

引用相关文件，提供背景信息。

# 好
> 参考 @src/auth/login.ts 的风格
> 创建类似的注册组件

# 差
> 创建一个注册组件

验证结果

让 Claude 验证自己的工作。

# 好
> 实现功能后运行测试确保正确

# 差
> 实现功能

实战示例

场景一：理解新代码库

# 刚接手项目时
> 这个项目是做什么的？
> 主入口在哪里？
> 有哪些主要模块？
> 数据库结构是怎样的？
> 如何运行测试？

场景二：功能开发完整流程

# 完整开发流程
> 我需要添加密码重置功能，创建一个实现计划
> 按照计划实现密码重置功能
> 为密码重置功能编写测试
> 运行测试确保一切正常
> 为这个功能创建提交

场景三：调试和修复

# 调试流程
> 用户使用特殊字符密码登录时出现 500 错误，请修复
> 调试这个错误：[错误栈]
> 检查是否有类似的问题
> 添加测试防止回归

场景四：代码重构

# 重构流程
> 将 @src/controllers/UserController.ts 重构为更模块化的结构
> 保持功能一致性
> 更新相关测试
> 确保所有测试通过
