使用模式详解

可靠性: A 来源: Claude Code 官方文档

更新时间: 2025-04-04 作者: Claude Code 教程团队

概述

Claude Code 提供多种使用模式，适应不同的工作场景和安全需求。理解这些模式可以帮助你更高效地使用 Claude Code。

核心概念：不同模式在安全性和效率之间提供不同的平衡，选择合适的模式可以提升工作效率。

默认模式

默认模式下，Claude 在执行每个操作前都会请求你的确认。

特点

  • 最安全 - 每个操作都需要确认
  • 可审查 - 可以查看每个操作的详情
  • 可修改 - 可以在执行前修改操作

适用场景

新项目

首次接触的项目，需要了解 Claude 的操作。

重要更改

关键代码或生产环境的修改。

学习阶段

学习 Claude Code 的工作方式。

确认选项

选项 快捷键 说明
允许 Enter 执行该操作
拒绝 Esc 取消该操作
修改 e 编辑操作内容后再执行
查看详情 v 查看完整的操作详情

自动接受模式

自动接受模式下，Claude 会自动执行操作，但仍会显示操作内容。

切换方式

# 按 Shift+Tab 切换自动接受模式
# 状态栏会显示 "⚡ auto-accept on"

特点

  • 高效率 - 无需手动确认每个操作
  • 可见性 - 仍然显示所有操作
  • 可回滚 - 可以撤销不需要的更改

适用场景

日常开发

熟悉的代码库，常规开发任务。

批量操作

需要执行大量相似操作的任务。

时间紧迫

需要快速完成任务的场景。

注意：使用自动接受模式时，建议定期检查 Claude 的操作，确保符合预期。

Plan Mode（规划模式）

Plan Mode 是 Claude Code 2.0 的杀手功能，将研究规划与代码执行分离。

激活方式

# 按两次 Shift+Tab 进入 Plan Mode
# 状态栏显示 "⏸ plan mode on"

工作原理

Plan Mode 限制

在 Plan Mode 下，Claude 只能使用只读工具：

  • Read - 读取文件
  • LS - 列出目录
  • Glob - 文件模式匹配
  • Grep - 内容搜索
  • Web 搜索 - 查找资料

所有修改操作（Edit、Write、Bash）都被阻止。

工作流程

# 传统工作流
提示 → 编码 → 发现 Bug → 修复 → 发布

# Plan Mode 工作流
提示 → 研究 → 规划 → 审查 → 执行 → 发布

使用示例

# 进入 Plan Mode（按两次 Shift+Tab）
> 构建带用户账户和分享功能的 TODO 应用。深度思考架构设计

# Claude 会花 5-10 分钟：
# - 研究最佳实践
# - 分析技术选型
# - 规划数据库模式
# - 设计 API 端点
# - 确定实现顺序

# 审查计划后，退出 Plan Mode 执行

核心价值

价值 说明
防止范围蔓延 提前发现需求问题
提早发现问题 在编码前发现集成问题
准确估算 更准确的时间估算
减少调试 减少后续调试时间
安全探索 不会意外破坏代码

YOLO 模式

YOLO（You Only Live Once）模式完全跳过权限检查，适合自动化场景。

启用方式

# 使用命令行参数
claude --dangerously-skip-permissions

# 或在会话中设置
> /permissions skip-all

特点

  • 最高效率 - 无任何确认步骤
  • 高风险 - 可能执行意外操作
  • 自动化 - 适合 CI/CD 场景

危险警告：YOLO 模式可能导致数据丢失或系统损坏。仅在完全信任 Claude 的环境中使用。

安全使用 YOLO 模式

# 推荐：结合 Docker 使用
docker run -it --rm \
  -v $(pwd):/workspace \
  claude-code \
  --dangerously-skip-permissions \
  "实现功能 X 并运行测试"

# 推荐：TDD 工作流
# 1. 先编写失败测试
claude "为功能 X 编写失败测试"
git commit -m "添加测试"

# 2. YOLO 模式实现
claude --dangerously-skip-permissions "让所有测试通过"

模式对比

模式 安全性 效率 适用场景
默认模式 ⭐⭐⭐⭐⭐ ⭐⭐ 新项目、重要更改
自动接受 ⭐⭐⭐⭐ ⭐⭐⭐⭐ 日常开发、批量操作
Plan Mode ⭐⭐⭐⭐⭐ ⭐⭐⭐ 复杂功能、架构设计
YOLO 模式 ⭐ ⭐⭐⭐⭐⭐ CI/CD、容器环境

最佳实践

灵活切换

根据任务性质灵活切换模式，不必一直使用同一种模式。

Git 保护

使用自动接受或 YOLO 模式前，先提交当前更改。

Plan Mode 优先

复杂任务先用 Plan Mode 规划，再执行。

容器隔离

YOLO 模式在 Docker 容器中使用更安全。

推荐工作流

# 1. 新项目：默认模式
claude
> 了解这个项目的结构

# 2. 复杂功能：Plan Mode
# 按 Shift+Tab 两次
> 设计用户认证系统

# 3. 日常开发：自动接受模式
# 按 Shift+Tab 一次
> 实现登录功能

# 4. CI/CD：YOLO 模式
claude --dangerously-skip-permissions "运行测试并修复失败"
