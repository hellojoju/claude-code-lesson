macOS 安装详解

可靠性: A 来源: Claude Code 官方文档

更新时间: 2025-04-04 作者: Claude Code 教程团队

系统要求

最低配置

操作系统 macOS 12 (Monterey) 或更高版本
芯片 Apple Silicon (M1/M2/M3) 或 Intel
Node.js v18.0.0 或更高版本
内存 4GB RAM（推荐 8GB+）
磁盘空间 500MB 可用空间

方法一：使用 Homebrew（推荐）

Homebrew 是 macOS 上最流行的包管理器，安装过程简单快捷。

安装 Homebrew（如未安装）

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

安装 Node.js

# 安装 Node.js 20
brew install node@20

# 或安装最新版
brew install node

安装 Claude Code

# 使用 npm 全局安装
npm install -g @anthropic-ai/claude-code

Apple Silicon 注意事项

如果你使用 Apple Silicon Mac，Homebrew 默认安装在 /opt/homebrew。确保 PATH 环境变量正确设置：

# 添加到 ~/.zshrc
eval "$(/opt/homebrew/bin/brew shellenv)"

方法二：原生安装器

使用官方安装脚本，无需预先安装 Node.js：

# 使用官方安装脚本
curl -fsSL https://claude.ai/install.sh | bash

优势：原生安装器会自动处理依赖，是最稳定且性能最优的方案。

卸载旧版本（如需要）

# 如果之前通过 npm 安装过，先卸载
npm uninstall -g @anthropic-ai/claude-code

# 然后使用原生安装器
curl -fsSL https://claude.ai/install.sh | bash

方法三：npm 安装

如果你已经安装了 Node.js，可以直接使用 npm：

使用 nvm 管理 Node.js

# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 重新加载 shell
source ~/.zshrc

# 安装 Node.js 20
nvm install 20
nvm use 20

# 安装 Claude Code
npm install -g @anthropic-ai/claude-code

使用官方 Node.js 安装包

  1. 访问 Node.js 官网
  2. 下载 LTS 版本（推荐 v20.x）
  3. 运行安装程序
  4. 执行 npm install -g @anthropic-ai/claude-code

验证安装

# 检查版本
claude --version

# 运行健康检查
claude doctor

# 首次启动
claude

健康检查输出示例

✓ Node.js version: v20.10.0
✓ npm version: 10.2.3
✓ Claude Code version: 1.0.0
✓ Configuration directory exists
✓ All checks passed!

首次登录

首次运行时，Claude Code 会引导你完成登录：

  1. 选择登录方式（浏览器或 API Key）
  2. 完成 Anthropic 账户授权
  3. 选择订阅计划（Pro 或 Max）

常见问题

权限错误？

不要使用 sudo npm install，改用原生安装器：

curl -fsSL https://claude.ai/install.sh | bash

命令找不到？

确保 npm 全局目录在 PATH 中：

# 检查 npm 全局目录
npm config get prefix

# 添加到 ~/.zshrc
export PATH="$(npm config get prefix)/bin:$PATH"

# 重新加载配置
source ~/.zshrc

Apple Silicon 兼容问题？

确保使用原生 ARM64 版本的 Node.js：

# 检查架构
node -p process.arch
# 应该输出 "arm64"

# 如果输出 "x64"，重新安装 ARM64 版本
brew reinstall node

Rosetta 问题？

如果你在 Intel 模式下运行终端，可能会遇到问题：

# 检查终端架构
uname -m
# 应该输出 "arm64"

# 如果输出 "x86_64"，请使用原生 ARM64 终端
