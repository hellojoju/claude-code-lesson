钩子概念详解

可靠性: A 来源: Claude Code 官方文档

更新时间: 2025-04-04 作者: Claude Code 教程团队

什么是钩子

钩子（Hook）是 Claude Code 的事件监听机制，允许在特定事件发生时执行自定义 Shell 命令。通过钩子，可以扩展 Claude Code 的功能，实现自动化工作流。

核心概念：如果将 Claude Code 的工作流程比作一条繁忙的高速公路，那么钩子就是沿途设置的智能交通信号灯，能够在预设的关键路口进行流量控制、安全检查和信息收集等操作。

核心组件

组件 说明 示例
事件 触发钩子的时机点 PreToolUse、PostToolUse
匹配器 决定是否执行钩子的条件 Bash、Edit、Write
处理器 事件触发时执行的 Shell 命令 Python 脚本、Shell 命令
上下文 传递给处理器的 JSON 数据 tool_name、tool_input
结果 处理器返回的操作指令 allow、deny、block

为什么需要钩子

钩子提供了确定性的控制机制，通过将规则和流程编码为应用程序级代码而非提示指令，能够保证命令按预期执行。

核心优势

自动化

自动执行重复性任务，无需每次提醒 Claude。

验证

在操作前进行验证检查，防止错误操作。

通知

发送通知和提醒，保持团队同步。

集成

与外部系统集成，扩展 Claude 能力。

典型应用场景

场景 具体功能 示例
智能通知 自动发送桌面通知 Claude 需要输入时发送通知
代码格式化 自动格式化代码 编辑 .ts 文件后运行 prettier
日志审计 记录所有操作 记录执行的 Bash 命令
质量反馈 代码质量检查 编辑后运行 linter 检查
安全控制 阻止敏感操作 禁止修改生产环境文件

钩子类型

Claude Code 支持多种钩子类型，每种类型代表不同的触发时机。

调用工具类钩子

PreToolUse

触发时机：在 Claude 执行任何工具之前

常见用途：权限检查、日志记录、预处理

{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{ "type": "command", "command": "python3 check.py" }]
    }]
  }
}

PostToolUse

触发时机：在 Claude 执行工具之后

常见用途：清理、验证、自动格式化

{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit",
      "hooks": [{ "type": "command", "command": "prettier --write $FILE" }]
    }]
  }
}

用户交互类钩子

UserPromptSubmit

触发时机：用户提交提示时（大模型处理前）

常见用途：输入验证、内容过滤、上下文注入

Notification

触发时机：Claude 发送通知时

常见用途：桌面通知、邮件提醒、Slack 集成

会话生命周期类钩子

SessionStart

触发时机：会话开始或恢复时

匹配器：startup、resume、clear、compact

常见用途：加载上下文、初始化环境

SessionEnd

触发时机：会话结束时

匹配器：clear、logout、prompt_input_exit、other

常见用途：清理临时文件、保存日志

任务完成类钩子

Stop

触发时机：主代理任务停止时

常见用途：完整性检查、自动提交

SubagentStop

触发时机：子代理任务停止时

常见用途：子任务验证、结果汇总

输入与输出

钩子输入（stdin）

钩子通过标准输入接收 JSON 数据：

{
  "session_id": "abc123",
  "transcript_path": "/Users/.../.claude/projects/.../session.jsonl",
  "cwd": "/Users/project",
  "hook_event_name": "PreToolUse",
  "tool_name": "Bash",
  "tool_input": {
    "command": "npm install",
    "description": "安装依赖"
  }
}

钩子输出

钩子可以通过两种方式输出：

方式一：退出状态码

退出码 含义 行为
0 成功 stdout 在 transcript 模式下展示
2 阻塞错误 stderr 反馈给 Claude 处理
其他 非阻塞错误 stderr 展示给用户，继续执行

方式二：结构化 JSON

{
  "continue": true,
  "stopReason": "string",
  "suppressOutput": false,
  "hookSpecificOutput": {
    "hookEventName": "PreToolUse",
    "permissionDecision": "allow" | "deny" | "ask",
    "permissionDecisionReason": "原因说明"
  }
}

权限决策

决策 功能 reason 展示对象
allow 绕过权限系统 仅向用户展示
deny 阻止工具调用 向 Claude 展示
ask 要求用户确认 仅向用户展示

配置方式

配置文件位置

配置类型 文件位置 适用范围
用户级 ~/.claude/settings.json 所有项目
项目级 .claude/settings.json 当前项目
本地配置 .claude/settings.local.json 个人本地（不提交）

配置示例

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "python3 $CLAUDE_PROJECT_DIR/.claude/hooks/validate_bash.py",
            "timeout": 30
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "prettier --write $FILE"
          }
        ]
      }
    ]
  }
}

在脚本中妥善处理错误，返回有意义的错误信息。

cat ~/.claude/logs/hooks.log

echo '{"tool_name": "Bash", "tool_input": {"command": "ls"}}' | python3 hook.py
