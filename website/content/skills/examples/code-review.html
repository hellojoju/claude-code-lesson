代码审查技能案例

来源说明：本文展示一个完整的代码审查技能实现。

可靠性评级：B 社区实践

技能概述

这个代码审查技能可以自动审查代码，发现潜在问题并提供改进建议。

功能特性

  • 多维度代码审查（安全、性能、风格、最佳实践）
  • 支持多种编程语言
  • 生成详细的审查报告
  • 提供具体的修复建议

技能配置

// skill.json
{
  "name": "code-review",
  "version": "2.0.0",
  "description": "智能代码审查技能",
  "author": "community",
  
  "triggers": {
    "manual": true,
    "keywords": ["审查", "review", "检查代码", "code review"],
    "auto": ["on-commit"]
  },
  
  "inputs": {
    "file": {
      "type": "file",
      "required": true,
      "description": "要审查的文件"
    },
    "focus": {
      "type": "string",
      "options": ["all", "security", "performance", "style", "best-practices"],
      "default": "all",
      "description": "审查重点"
    },
    "severity": {
      "type": "string",
      "options": ["all", "high", "medium", "low"],
      "default": "all",
      "description": "问题严重级别"
    }
  },
  
  "outputs": {
    "report": {
      "type": "markdown",
      "description": "审查报告"
    },
    "issues": {
      "type": "json",
      "description": "问题列表（JSON 格式）"
    }
  }
}

你是一位资深的代码审查专家。请对以下代码进行全面审查。

## 文件信息
- 文件：{{file.path}}
- 语言：{{file.language}}
- 行数：{{file.lines}}

## 代码内容
```
{{file.content}}
```

## 审查重点
{{#if (eq focus "security")}}

- SQL 注入
- XSS 攻击
- 敏感信息泄露

{{#if (eq focus "performance")}}

{{#if (eq focus "style")}}

使用方法

基本使用

# 审查单个文件
> /skill run code-review --file src/index.js

# 指定审查重点
> /skill run code-review --file src/api.js --focus security

# 只看高优先级问题
> /skill run code-review --file src/index.js --severity high

批量审查

# 审查所有 JS 文件
> /glob "*.js" | /skill run code-review

# 审查最近修改的文件
> /git diff --name-only | /skill run code-review

自然语言触发

# 使用关键词触发
> 审查 src/index.js 的代码质量
> 检查 src/api.js 的安全问题
> review this code for performance issues

输出示例

# 代码审查报告

## 文件：src/api.js

### 🔴 高优先级问题

#### 1. SQL 注入风险（第 25 行）
```javascript
const query = `SELECT * FROM users WHERE id = ${userId}`;
```
**问题**：直接拼接用户输入到 SQL 语句，存在 SQL 注入风险。

**修复建议**：使用参数化查询
```javascript
const query = 'SELECT * FROM users WHERE id = ?';
db.query(query, [userId]);
```

### 🟡 中优先级问题

#### 2. 未处理的 Promise（第 42 行）
```javascript
fetchData().then(data => process(data));
```
**问题**：Promise 没有错误处理。

**修复建议**：添加 catch 处理
```javascript
fetchData()
  .then(data => process(data))
  .catch(error => console.error('Error:', error));
```

### 🟢 低优先级建议

#### 3. 使用 const 替代 let（第 15 行）
```javascript
let API_URL = 'https://api.example.com';
```
**建议**：如果不会重新赋值，使用 const
```javascript
const API_URL = 'https://api.example.com';
```

### ✅ 优点
- 函数命名清晰
- 有适当的注释
- 错误处理完善（大部分）

### 📊 总结
- 问题总数：3
- 高优先级：1
- 中优先级：1
- 低优先级：1
