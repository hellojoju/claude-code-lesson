测试生成技能案例

来源说明：本文展示一个完整的测试生成技能实现。

可靠性评级：B 社区实践

技能概述

这个测试生成技能可以自动为代码生成单元测试，支持多种测试框架。

功能特性

  • 支持多种测试框架（Jest、Vitest、Mocha 等）
  • 自动识别函数边界和参数类型
  • 生成边界条件和异常测试
  • 支持 React 组件测试

## 测试要求

### 1. 测试覆盖
- 正常情况测试
- 边界条件测试
- 异常情况测试
- 空值/undefined 测试

### 2. 测试结构
{{#if (eq framework "jest")}}
```javascript
describe('功能模块', () => {
  describe('具体功能', () => {
    it('应该...', () => {
      // arrange
      // act
      // assert
    });
  });
});
```
{{/if}}

{{#if (eq framework "vitest")}}
```typescript
import { describe, it, expect } from 'vitest';

describe('功能模块', () => {
  it('应该...', () => {
    expect(true).toBe(true);
  });
});
```
{{/if}}

### 3. 测试原则
- 每个测试只验证一个行为
- 使用描述性的测试名称
- 遵循 AAA 模式（Arrange-Act-Assert）
- 使用有意义的测试数据

### 4. Mock 规范
- 外部依赖需要 mock
- 使用清晰的 mock 名称
- 验证 mock 调用

1. 必要的导入

3. 测试用例

使用方法

基本使用

# 生成单元测试
> /skill run test-gen --file src/utils.js

# 指定测试框架
> /skill run test-gen --file src/utils.js --framework vitest

# 生成集成测试
> /skill run test-gen --file src/api.js --type integration

React 组件测试

# 生成 React 组件测试
> /skill run test-gen --file src/components/Button.tsx --framework vitest

# 输出示例
// Button.test.tsx
import { describe, it, expect } from 'vitest';
import { render, screen } from '@testing-library/react';
import { Button } from './Button';

describe('Button', () => {
  it('should render with correct text', () => {
    render(<Button>Click me</Button>);
    expect(screen.getByText('Click me')).toBeInTheDocument();
  });

  it('should call onClick when clicked', () => {
    const handleClick = vi.fn();
    render(<Button onClick={handleClick}>Click me</Button>);
    screen.getByText('Click me').click();
    expect(handleClick).toHaveBeenCalledTimes(1);
  });

  it('should be disabled when disabled prop is true', () => {
    render(<Button disabled>Click me</Button>);
    expect(screen.getByText('Click me')).toBeDisabled();
  });
});

源文件

// src/utils/formatDate.ts
export function formatDate(date: Date | string, format: string = 'YYYY-MM-DD'): string {
  if (!date) {
    throw new Error('Date is required');
  }
  
  const d = typeof date === 'string' ? new Date(date) : date;
  
  if (isNaN(d.getTime())) {
    throw new Error('Invalid date');
  }
  
  return format
    .replace('YYYY', d.getFullYear().toString())
    .replace('MM', (d.getMonth() + 1).toString().padStart(2, '0'))
    .replace('DD', d.getDate().toString().padStart(2, '0'));
}

生成的测试

// src/utils/formatDate.test.ts
import { describe, it, expect, beforeEach } from 'vitest';
import { formatDate } from './formatDate';

describe('formatDate', () => {
  describe('正常情况', () => {
    it('应该正确格式化日期对象', () => {
      const date = new Date('2026-04-04');
      expect(formatDate(date)).toBe('2026-04-04');
    });

    it('应该正确格式化日期字符串', () => {
      expect(formatDate('2026-04-04')).toBe('2026-04-04');
    });

    it('应该支持自定义格式', () => {
      const date = new Date('2026-04-04');
      expect(formatDate(date, 'YYYY/MM/DD')).toBe('2026/04/04');
    });
  });

  describe('边界条件', () => {
    it('应该正确处理月份边界', () => {
      const date = new Date('2026-01-01');
      expect(formatDate(date)).toBe('2026-01-01');
    });

    it('应该正确处理年末日期', () => {
      const date = new Date('2026-12-31');
      expect(formatDate(date)).toBe('2026-12-31');
    });
  });

  describe('异常情况', () => {
    it('应该在日期为空时抛出错误', () => {
      expect(() => formatDate(null as any)).toThrow('Date is required');
    });

    it('应该在日期无效时抛出错误', () => {
      expect(() => formatDate('invalid-date')).toThrow('Invalid date');
    });
  });
});
