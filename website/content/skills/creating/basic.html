创建基础技能

来源说明：本文内容基于 Claude Code 官方文档和社区最佳实践整理。

可靠性评级：A 官方文档 + 社区验证

快速开始

创建第一个技能

# 创建技能目录
mkdir -p ~/.claude/skills/hello-world

# 创建技能配置
cat > ~/.claude/skills/hello-world/skill.json << 'EOF'
{
  "name": "hello-world",
  "version": "1.0.0",
  "description": "简单的问候技能",
  "triggers": {
    "manual": true
  }
}
EOF

# 创建提示词文件
cat > ~/.claude/skills/hello-world/prompt.md << 'EOF'
你是一个友好的助手。请向用户问好，并简单介绍你能做什么。
EOF

使用技能

# 安装技能
> /skill install ~/.claude/skills/hello-world

# 运行技能
> /skill run hello-world

技能结构详解

必需文件

my-skill/
├── skill.json    # 技能配置（必需）
└── prompt.md     # 主提示词（必需）

可选文件

my-skill/
├── templates/    # 模板文件
├── examples/     # 示例文件
├── lib/          # 辅助代码
└── tests/        # 测试文件

技能配置

基础配置

{
  "name": "my-skill",
  "version": "1.0.0",
  "description": "技能描述",
  "author": "作者名",
  "triggers": {
    "manual": true
  }
}

完整配置

{
  "name": "code-review",
  "version": "1.0.0",
  "description": "代码审查技能",
  "author": "your-name",
  "license": "MIT",
  
  "triggers": {
    "manual": true,
    "auto": ["on-save"],
    "keywords": ["审查", "review", "检查代码"]
  },
  
  "inputs": {
    "file": {
      "type": "file",
      "required": true,
      "description": "要审查的文件"
    },
    "focus": {
      "type": "string",
      "options": ["security", "performance", "style"],
      "default": "security",
      "description": "审查重点"
    }
  },
  
  "outputs": {
    "report": {
      "type": "markdown",
      "description": "审查报告"
    }
  },
  
  "dependencies": {
    "skills": ["analyze"],
    "packages": []
  }
}

提示词文件

基本提示词

# prompt.md

你是一个代码审查专家。请审查用户提供的代码，关注以下方面：

1. 代码质量
2. 安全问题
3. 性能考虑
4. 最佳实践

请提供具体的改进建议。

带变量的提示词

# prompt.md

你是一个 {{focus}} 领域的专家。

请审查以下代码：
{{file}}

审查重点：{{focus}}

请提供详细的审查报告。

使用变量

# 传递变量
> /skill run code-review --file src/index.js --focus security

触发器配置

手动触发

{
  "triggers": {
    "manual": true
  }
}

关键词触发

{
  "triggers": {
    "manual": true,
    "keywords": ["审查", "review", "检查代码"]
  }
}

事件触发

{
  "triggers": {
    "auto": ["on-save", "on-commit"]
  }
}

条件触发

{
  "triggers": {
    "auto": ["on-save"],
    "conditions": {
      "filePattern": "*.js",
      "exclude": ["*.test.js"]
    }
  }
}

完整示例

示例：代码格式化技能

// skill.json
{
  "name": "format-code",
  "version": "1.0.0",
  "description": "代码格式化技能",
  "triggers": {
    "manual": true,
    "keywords": ["格式化", "format", "美化代码"]
  },
  "inputs": {
    "file": {
      "type": "file",
      "required": true
    },
    "style": {
      "type": "string",
      "options": ["standard", "prettier", "airbnb"],
      "default": "prettier"
    }
  }
}
# prompt.md

你是一个代码格式化专家。请按照 {{style}} 风格格式化以下代码：

{{file}}

要求：
1. 保持代码功能不变
2. 遵循 {{style}} 代码规范
3. 添加适当的空格和换行
4. 优化代码结构

请输出格式化后的代码。
