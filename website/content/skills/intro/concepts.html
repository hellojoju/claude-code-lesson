技能系统概念

可靠性: A 来源: Claude Code 官方文档

更新时间: 2025-04-04 作者: Claude Code 教程团队

什么是技能系统

技能系统是 Claude Code 的核心扩展机制，允许用户定义可复用的能力模块。每个技能封装了特定的任务处理逻辑，可以被 Claude 在需要时自动调用。

核心价值：技能系统让 Claude Code 从一个通用助手变成可以针对特定任务进行专业化处理的智能代理。

技能 vs 命令

特性 技能 命令
触发方式 自动或手动 手动调用
复杂度 可包含复杂逻辑 通常是简单指令
上下文 可访问完整上下文 有限的上下文
复用性 高度可复用 一次性使用
可定制性 完全可定制 有限定制

核心优势

可复用

一次创建，到处使用，避免重复工作。

可组合

多个技能可以组合使用，处理复杂任务。

可共享

技能可以在团队间共享，提高协作效率。

自动触发

满足条件时自动执行，无需手动调用。

技能架构

系统组成

技能系统
├── 技能注册中心
│   ├── 内置技能
│   └── 自定义技能
├── 技能执行引擎
│   ├── 上下文构建
│   ├── 技能调用
│   └── 结果处理
└── 技能管理器
    ├── 安装/卸载
    ├── 启用/禁用
    └── 更新

技能存储位置

~/.claude/skills/           # 用户级技能
.project/skills/           # 项目级技能
~/.claude/builtin-skills/  # 内置技能

技能生命周期

1. 注册阶段
   - 加载 skill.json
   - 验证配置
   - 注册触发器

2. 触发阶段
   - 检测触发条件
   - 构建执行上下文
   - 准备输入参数

3. 执行阶段
   - 加载提示词模板
   - 执行技能逻辑
   - 处理输出结果

4. 清理阶段
   - 释放资源
   - 记录日志
   - 更新状态

技能类型

按功能分类

类型 说明 示例
代码生成 生成代码片段 组件生成、API 生成、脚手架
代码分析 分析代码质量 代码审查、复杂度分析、安全检查
测试生成 生成测试代码 单元测试、集成测试、E2E 测试
文档生成 生成文档 API 文档、README、注释生成
重构辅助 辅助代码重构 提取函数、重命名、移动文件

按触发方式分类

手动触发

用户显式调用，如 /skill run code-review

自动触发

满足条件时自动执行，如保存文件后自动格式化

事件触发

响应特定事件，如提交代码前自动检查

技能结构

技能文件结构

my-skill/
├── skill.json       # 技能配置
├── prompt.md        # 主提示词
├── templates/       # 模板文件
│   └── component.tsx
├── examples/        # 示例文件
│   └── example1.ts
└── lib/            # 辅助库
    └── utils.ts

技能配置文件

{
  "name": "code-review",
  "version": "1.0.0",
  "description": "代码审查技能",
  "author": "your-name",
  "triggers": {
    "manual": true,
    "auto": ["on-save", "on-commit"]
  },
  "inputs": {
    "file": {
      "type": "file",
      "required": true,
      "description": "要审查的文件"
    },
    "focus": {
      "type": "string",
      "options": ["security", "performance", "style"],
      "default": "security"
    }
  },
  "outputs": {
    "report": {
      "type": "markdown",
      "description": "审查报告"
    }
  }
}

基本使用

查看可用技能

# 列出所有技能
> /skill list

# 查看特定技能详情
> /skill info code-review

# 搜索技能
> /skill search test

使用技能

# 手动调用技能
> /skill run code-review --file src/index.js

# 使用自然语言触发
> 审查 src/index.js 的代码质量
# Claude 会自动识别并调用 code-review 技能

# 带参数调用
> /skill run test-gen --file src/utils.ts --type unit

管理技能

# 安装技能
> /skill install /path/to/skill

# 从 Git 安装
> /skill install https://github.com/user/claude-skill.git

# 启用/禁用技能
> /skill enable code-review
> /skill disable code-review

# 更新技能
> /skill update code-review

# 卸载技能
> /skill uninstall code-review

常见问题

技能触发冲突？

当多个技能可能同时触发时，Claude 会根据上下文选择最合适的技能。你可以通过显式调用来避免冲突：

# 显式指定技能
> /skill run code-review --file src/index.ts

技能执行失败？

检查技能配置和输入参数是否正确：

# 查看技能详情
> /skill info skill-name

# 查看执行日志
> /skill logs skill-name
