记忆系统最佳实践

来源说明：本文内容基于 Claude Code 社区最佳实践整理。

可靠性评级：B 社区验证

记忆组织

分类原则

  • 按作用域分类：项目级、用户级、全局级
  • 按内容分类：配置、知识、决策、解决方案
  • 按时间分类：永久、临时、会话级

标签规范

# 推荐的标签分类

## 内容类型
config      - 配置信息
knowledge   - 知识点
decision    - 决策记录
solution    - 解决方案
convention  - 规范约定

## 技术领域
react       - React 相关
typescript  - TypeScript 相关
api         - API 相关
security    - 安全相关
performance - 性能相关

## 优先级
important   - 重要记忆
temporary   - 临时记忆
deprecated  - 已废弃

记忆编写

编写原则

  • 简洁明了：一句话说清楚要点
  • 结构化：使用一致的格式
  • 可搜索：包含关键词
  • 可操作：提供具体信息

好的记忆示例

# 好的记忆
> /memory save --type project --tags "config,api" "API 基础地址: https://api.example.com/v1，认证: Bearer Token"

> /memory save --type global --tags "solution,cors" "解决 CORS: 在 vite.config.ts 的 server.proxy 中配置代理"

# 不好的记忆
> /memory save "API 配置"  # 太模糊
> /memory save "记得要用那个东西"  # 不明确

记忆模板

# 配置记忆模板

# 解决方案模板

# 决策记录模板

记忆维护

定期清理

# 定期执行清理

# 清理过期记忆
> /memory clear --expired

# 清理临时记忆
> /memory clear --tags "temporary"

# 清理废弃记忆
> /memory clear --tags "deprecated"

记忆更新

# 当信息变更时更新记忆

# 更新配置
> /memory update mem-001 "API 基础地址: https://new-api.example.com/v2"

# 标记废弃
> /memory tag mem-001 --add "deprecated"

# 创建新记忆替代
> /memory save --type project --tags "config,api" "API 基础地址: https://new-api.example.com/v2"

记忆审计

# 定期审计记忆

# 查看统计
> /memory stats

# 检查重复
> /memory check --duplicates

# 检查过期
> /memory check --expired

安全考虑

敏感信息处理

安全警告：

  • 不要在记忆中存储密码、密钥等敏感信息
  • 使用环境变量引用敏感配置
  • 定期审查记忆中的敏感信息

安全示例

# 不安全
> /memory save "数据库密码: mypassword123"  # ❌ 不要这样做

# 安全
> /memory save "数据库连接: 使用环境变量 DATABASE_URL"  # ✓ 正确做法

# 引用环境变量
> /memory save --type project --tags "config" "API Key: 使用环境变量 VITE_API_KEY"

团队协作

共享项目记忆

# 将项目记忆纳入版本控制
cd .project
git add memory.json
git commit -m "docs: update project memory"

# 团队成员同步
git pull
# Claude 会自动加载更新后的记忆

团队规范记忆

# 团队编码规范
> /memory save --type project --tags "team,convention" "分支命名: feature/xxx, fix/xxx, hotfix/xxx"
> /memory save --type project --tags "team,convention" "提交格式: type(scope): description"
> /memory save --type project --tags "team,convention" "代码审查: PR 需要至少一人 approve"

# 团队工作流
> /memory save --type project --tags "team,workflow" "开发流程: feature branch → PR → code review → merge"
> /memory save --type project --tags "team,workflow" "发布流程: develop → release → main"

性能优化

记忆数量控制

# 建议的记忆数量
项目记忆: < 50 条
用户记忆: < 30 条
全局记忆: < 100 条

# 查看记忆数量
> /memory stats

优化建议

  • 合并相似记忆
  • 删除过时记忆
  • 使用标签分类
  • 设置合理的优先级

合并相似记忆

# 合并前
> /memory save "使用 React 18"
> /memory save "使用 TypeScript"
> /memory save "使用 Vite"

# 合并后
> /memory save --type project --tags "config,tech" "技术栈: React 18 + TypeScript + Vite"

最佳实践检查清单

创建记忆时

  • 选择正确的记忆类型
  • 添加适当的标签
  • 使用清晰简洁的描述
  • 不包含敏感信息

维护记忆时

  • 定期清理过期记忆
  • 更新变更的信息
  • 合并重复记忆
  • 审查敏感信息

团队协作时

  • 项目记忆纳入版本控制
  • 建立团队记忆规范
  • 定期同步记忆更新
  • 新成员培训记忆使用
