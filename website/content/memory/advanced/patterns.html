记忆模式

来源说明：本文内容基于 Claude Code 社区最佳实践整理。

可靠性评级：B 社区验证

记忆模式概述

记忆模式是一套经过验证的记忆使用方法，帮助用户更有效地组织和管理记忆。

常见模式

  • 项目配置模式：存储项目级配置信息
  • 知识库模式：构建个人知识库
  • 决策记录模式：记录重要决策及其原因
  • 问题解决模式：保存解决方案供复用

项目配置模式

模式描述

将项目相关的配置信息集中存储在项目记忆中，便于 Claude 理解项目背景。

实现方式

# 项目初始化时创建配置记忆
> /memory save --type project --tags "config,init" "项目名称: my-app"
> /memory save --type project --tags "config,tech" "技术栈: React 18 + TypeScript + Vite"
> /memory save --type project --tags "config,build" "构建工具: Vite"
> /memory save --type project --tags "config,test" "测试框架: Vitest + Testing Library"

# 目录结构记忆
> /memory save --type project --tags "structure" "源码目录: src/"
> /memory save --type project --tags "structure" "组件目录: src/components/"
> /memory save --type project --tags "structure" "页面目录: src/pages/"

# API 配置记忆
> /memory save --type project --tags "api" "API 基础地址: https://api.example.com/v1"
> /memory save --type project --tags "api" "认证方式: JWT Bearer Token"

使用效果

# Claude 会自动使用这些记忆
> 创建一个新组件
# Claude 知道使用 TypeScript 和 React 18，放在 src/components/

> 调用用户 API
# Claude 知道 API 基础地址和认证方式

问题解决模式

模式描述

保存遇到的问题和解决方案，避免重复解决相同问题。

实现方式

# 问题解决记录
> /memory save --type global --tags "solution,cors" "
问题: CORS 错误
场景: 前端调用后端 API 时出现跨域错误
解决方案: 在 vite.config.ts 中配置 proxy
代码:
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:3000',
        changeOrigin: true
      }
    }
  }
"

# Bug 修复记录
> /memory save --type project --tags "solution,bug" "
问题: React useEffect 无限循环
原因: 依赖数组中使用了每次渲染都会变化的对象
解决方案: 使用 useMemo 缓存对象，或提取到组件外部
"

# 性能问题记录
> /memory save --type global --tags "solution,performance" "
问题: React 组件渲染慢
诊断: 使用 React DevTools Profiler 分析
解决方案: 
1. 使用 React.memo 避免不必要渲染
2. 使用 useMemo/useCallback 缓存计算结果
3. 虚拟化长列表
"

模式描述

实现方式

# 任务上下文

当前任务: 重构用户模块
目标: 将 Class 组件改为函数组件
进度: 已完成 50%
下一步: 重构 UserProfile 组件
"

相关文件: src/hooks/useUser.ts, src/types/user.ts
注意事项: 保持 API 兼容性
