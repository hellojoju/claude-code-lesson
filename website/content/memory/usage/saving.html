保存记忆详解

来源说明：本文内容基于 Claude Code 官方文档和实际使用案例整理。

可靠性评级：A 官方文档 + 实践验证

基本保存方法

自然语言保存

# 使用自然语言保存
> 记住：这个项目使用 React 18

> 记住我的偏好：使用 2 空格缩进

> 记住重要：API 密钥存储在 .env 文件中

命令保存

# 使用命令保存
> /memory save "项目使用 React 18"

# 指定类型保存
> /memory save --type project "技术栈: React + TypeScript"
> /memory save --type user "偏好: 深色主题"
> /memory save --type global "最佳实践: 使用函数组件"

保存选项

指定类型

# 项目记忆
> /memory save --type project "内容"

# 用户记忆
> /memory save --type user "内容"

# 全局记忆
> /memory save --type global "内容"

# 会话记忆
> /memory save --type session "内容"

设置标签

# 添加标签便于搜索
> /memory save --tags "config,api" "API 基础地址: https://api.example.com"

# 多个标签
> /memory save --tags "react,components,style" "组件样式使用 CSS Modules"

设置优先级

# 设置优先级（1-100）
> /memory save --priority 100 "关键配置: 数据库连接字符串"

# 高优先级记忆会被优先使用
> /memory save --priority 50 "一般配置: 日志级别"

设置过期时间

# 设置过期时间
> /memory save --expires 7d "临时配置: 测试环境 API"

# 过期时间格式
--expires 1h    # 1 小时
--expires 1d    # 1 天
--expires 1w    # 1 周
--expires 1m    # 1 月

结构化保存

JSON 格式

# 保存 JSON 格式记忆
> /memory save --format json '{
  "name": "项目配置",
  "tech_stack": ["React", "TypeScript", "Vite"],
  "api": {
    "base_url": "https://api.example.com",
    "version": "v1"
  }
}'

键值对格式

# 保存键值对
> /memory set project.name "我的项目"
> /memory set project.version "1.0.0"
> /memory set api.base_url "https://api.example.com"

# 读取键值
> /memory get project.name

列表格式

# 保存列表
> /memory push tech_stack "React"
> /memory push tech_stack "TypeScript"
> /memory push tech_stack "Vite"

# 查看列表
> /memory get tech_stack

自动保存

自动保存触发条件

Claude 会自动识别并保存以下类型的信息：

  • 重要的架构决策
  • 项目配置变更
  • 用户明确表达的偏好
  • 关键的问题解决方案

配置自动保存

// ~/.claude/config.json
{
  "memory": {
    "autoSave": true,
    "autoSaveCategories": [
      "architecture",
      "configuration",
      "preferences",
      "solutions"
    ],
    "maxAutoSavePerSession": 10
  }
}

查看自动保存的记忆

# 查看自动保存的记忆
> /memory list --source auto

# 查看最近保存的记忆
> /memory list --recent

批量保存

从文件导入

# 从 JSON 文件导入
> /memory import memories.json

# 从 Markdown 文件导入
> /memory import notes.md --format markdown

批量保存示例

// memories.json
[
  {
    "type": "project",
    "content": "技术栈: React 18 + TypeScript",
    "tags": ["config", "tech"]
  },
  {
    "type": "project",
    "content": "构建工具: Vite",
    "tags": ["config", "build"]
  },
  {
    "type": "user",
    "content": "代码风格: 单引号，无分号",
    "tags": ["style", "preferences"]
  }
]

实际应用示例

项目初始化时保存

# 保存项目基本信息
> /memory save --type project --tags "init" "项目名称: my-app"
> /memory save --type project --tags "init" "创建日期: 2026-04-04"
> /memory save --type project --tags "init" "技术栈: React + TypeScript + Vite"

# 保存项目结构
> /memory save --type project --tags "structure" "组件目录: src/components"
> /memory save --type project --tags "structure" "页面目录: src/pages"
> /memory save --type project --tags "structure" "工具目录: src/utils"

开发过程中保存

# 保存 API 配置
> /memory save --type project --tags "api" "API 基础地址: https://api.example.com/v1"
> /memory save --type project --tags "api" "认证方式: JWT Bearer Token"

# 保存重要决策
> /memory save --type project --tags "decision" "状态管理: 使用 Zustand，不使用 Redux"
> /memory save --type project --tags "decision" "样式方案: Tailwind CSS + CSS Modules"

# 保存解决方案
> /memory save --type global --tags "solution" "解决 CORS: 在 vite.config.ts 中配置 proxy"

团队协作保存

# 保存团队规范
> /memory save --type project --tags "team,convention" "分支命名: feature/xxx, fix/xxx"
> /memory save --type project --tags "team,convention" "提交格式: Conventional Commits"
> /memory save --type project --tags "team,convention" "代码审查: 必须至少一人 approve"
