记忆系统概念

可靠性: A 来源: Claude Code 官方文档

更新时间: 2025-04-04 作者: Claude Code 教程团队

什么是记忆系统

记忆系统是 Claude Code 的核心特性之一，它允许 Claude 在不同会话之间保持上下文和知识。这意味着 Claude 可以"记住"你的偏好、项目信息和重要决策，从而提供更连贯和个性化的帮助。

核心价值：记忆系统让 Claude Code 从一个"每次都要重新解释"的助手，变成一个"了解你和你的项目"的智能伙伴。

为什么需要记忆系统？

上下文连续性

跨会话保持项目背景，无需重复解释。

个性化体验

记住用户偏好和习惯，提供定制化服务。

知识积累

保存重要决策和解决方案，构建知识库。

效率提升

避免重复解释相同信息，提高工作效率。

CLAUDE.md 文件

CLAUDE.md 是 Claude Code 的"长期记忆"，在项目根目录创建这个文件，Claude 每次启动都会自动加载，无需重复说明项目规范。

层级结构

CLAUDE.md 支持层级结构，优先级从高到低：

企业级（system-wide）
    ↓
用户级（~/.claude/CLAUDE.md）
    ↓
项目级（./CLAUDE.md）
    ↓
本地级（./CLAUDE.local.md，不提交到 git）
    ↓
模块级（./src/auth/CLAUDE.md）

典型 CLAUDE.md 内容

# 项目名称

## 技术栈
- Next.js 14 + TypeScript
- Tailwind CSS + shadcn/ui
- Prisma + PostgreSQL

## 编码规范
- 使用 TypeScript strict mode
- 遵循现有 ESLint 配置
- 组件放 src/components/
- 测试放源文件旁边用 .test.ts

## 架构说明
- 前端用 Next.js App Router
- 后端 API 路由在 app/api/
- 数据库操作用 Prisma Client

## 开发命令
- pnpm dev - 启动开发服务器
- pnpm test - 运行测试
- pnpm lint - 代码检查

## 禁止读取的目录
- node_modules
- dist
- .git
- .next

关键技巧

  • 保持 CLAUDE.md 在 5000 token 以内
  • 将非核心内容拆分到 docs/ 目录并通过 @~/coding-standards.md 引用
  • 明确指定禁止读取的目录以节省 token
  • 定期更新活跃功能和 TODO 列表

记忆系统架构

系统组成

记忆系统
├── 短期记忆 (Session Memory)
│   └── 当前会话的临时信息
├── 长期记忆 (Long-term Memory)
│   ├── 项目记忆 (CLAUDE.md)
│   ├── 用户记忆 (~/.claude/)
│   └── 全局记忆
└── 工作记忆 (Working Memory)
    └── 当前任务的上下文

记忆存储位置

~/.claude/
├── CLAUDE.md              # 用户级记忆
├── settings.json          # 用户设置
├── memory/
│   ├── global.json        # 全局记忆
│   └── user.json          # 用户偏好记忆
└── projects/
    ├── project-a/
    │   ├── CLAUDE.md      # 项目记忆
    │   └── memory.json    # 项目会话记忆
    └── project-b/
        └── CLAUDE.md

记忆类型

类型 作用域 持久性 用途
项目记忆 当前项目 永久 项目配置、架构决策、编码规范
用户记忆 当前用户 永久 个人偏好、常用设置、工作习惯
会话记忆 当前会话 临时 对话上下文、临时决策
全局记忆 所有项目 永久 通用知识、最佳实践

记忆优先级

当多个记忆类型包含相同信息时，优先级如下：

项目记忆 > 用户记忆 > 全局记忆

# 示例：
# 全局记忆：使用 4 空格缩进
# 用户记忆：使用 2 空格缩进
# 项目记忆：使用 tab 缩进
# 最终结果：使用 tab 缩进（项目记忆优先级最高）

工作原理

记忆流程

1. 用户输入
      ↓
2. 记忆检索
   - 加载 CLAUDE.md
   - 加载相关记忆
   - 构建上下文
      ↓
3. Claude 处理
   - 结合记忆生成响应
      ↓
4. 记忆更新
   - 保存新信息
   - 更新相关记忆

上下文管理

压缩

使用 /compact 压缩历史，保留关键信息。

清除

使用 /clear 清除当前会话上下文。

监控

使用 /context 查看上下文使用情况。

基本使用

保存记忆

# 自然语言保存记忆
> 记住：这个项目使用 TypeScript 和 React
> 记住我的偏好：使用 2 空格缩进
> 记住：每次提交前运行 npm test

# 查看记忆
> /memory list

使用记忆

# Claude 会自动使用相关记忆
> 创建一个新组件
# Claude 会记住你使用 TypeScript 和 React

> 格式化这段代码
# Claude 会记住你使用 2 空格缩进

管理记忆

# 列出所有记忆
> /memory list

# 搜索记忆
> /memory search "TypeScript"

# 删除记忆
> /memory delete [记忆ID]

# 清除项目记忆
> /memory clear --project

# 导出记忆
> /memory export > memory-backup.json

# 导入记忆
> /memory import < memory-backup.json

记忆命令列表

命令 说明
/memory save 保存记忆
/memory list 列出记忆
/memory search 搜索记忆
/memory delete 删除记忆
/memory clear 清除记忆
/memory export 导出记忆
/memory import 导入记忆

最佳实践

使用 CLAUDE.md

将项目级记忆写在 CLAUDE.md 中，而不是每次会话都说明。

保持精简

CLAUDE.md 控制在 5000 token 以内，避免信息过载。

定期更新

项目变化时及时更新 CLAUDE.md，保持信息准确。

版本控制

将 CLAUDE.md 纳入 Git，方便团队共享。

典型应用场景

项目配置记忆

# 保存项目技术栈
> 记住：这个项目使用 Next.js 14、Tailwind CSS、Prisma

# 保存编码规范
> 记住：函数命名使用 camelCase，组件使用 PascalCase

# 保存架构决策
> 记住：API 路由放在 app/api/ 目录下

个人偏好记忆

# 保存代码风格
> 记住我的偏好：使用单引号，不使用分号

# 保存常用配置
> 记住：测试框架使用 Jest，断言库使用 Testing Library

# 保存工作习惯
> 记住：每次提交前运行 lint 和 test

常见问题

记忆太多导致上下文过满？

定期清理不需要的记忆，将重要信息写入 CLAUDE.md：

# 清理临时记忆
> /memory clear --session

# 将重要信息写入 CLAUDE.md
> 把项目规范写入 CLAUDE.md

团队成员记忆不一致？

将项目级记忆写入 CLAUDE.md 并提交到 Git，确保团队共享：

# 项目级 CLAUDE.md 会被 Git 跟踪
git add CLAUDE.md
git commit -m "更新项目记忆"
git push
