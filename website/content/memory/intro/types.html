记忆类型详解

来源说明：本文内容基于 Claude Code 官方文档和源码分析整理。

可靠性评级：A 官方文档 + 源码验证

项目记忆

概述

项目记忆与特定项目绑定，存储项目相关的配置、架构决策和开发规范。

存储位置

.project/memory.json       # 项目级记忆
~/.claude/memory/projects/[project-hash]/memory.json  # 用户级项目记忆

使用场景

# 保存项目技术栈
> /memory save --type project "技术栈: React 18 + TypeScript + Vite"

# 保存架构决策
> /memory save --type project "架构: 采用微前端架构，主应用使用 qiankun"

# 保存编码规范
> /memory save --type project "规范: 组件放在 src/components，页面放在 src/pages"

# 保存重要配置
> /memory save --type project "API 基础地址: https://api.example.com/v1"

记忆结构

{
  "id": "proj-001",
  "type": "project",
  "project": "my-project",
  "created": "2026-04-04T10:00:00Z",
  "updated": "2026-04-04T10:30:00Z",
  "content": {
    "tech_stack": ["React", "TypeScript", "Vite"],
    "architecture": "micro-frontend",
    "conventions": {
      "components": "src/components",
      "pages": "src/pages"
    }
  },
  "tags": ["config", "architecture"],
  "priority": 10
}

用户记忆

概述

用户记忆存储个人偏好、常用设置和工作习惯，跨项目共享。

存储位置

~/.claude/memory/user.json

使用场景

# 保存代码风格偏好
> /memory save --type user "代码风格: 使用单引号，无分号，2空格缩进"

# 保存常用工具
> /memory save --type user "测试框架: Jest + Testing Library"

# 保存工作习惯
> /memory save --type user "提交规范: 使用 Conventional Commits"

# 保存个人信息
> /memory save --type user "作者信息: name: 张三, email: zhangsan@example.com"

记忆结构

{
  "id": "user-001",
  "type": "user",
  "created": "2026-04-04T10:00:00Z",
  "updated": "2026-04-04T10:30:00Z",
  "content": {
    "code_style": {
      "quotes": "single",
      "semicolons": false,
      "indent": 2
    },
    "tools": {
      "test_framework": "Jest",
      "assertion_library": "Testing Library"
    },
    "commit_convention": "Conventional Commits"
  },
  "tags": ["preferences", "style"],
  "priority": 5
}

会话记忆

概述

会话记忆仅在当前会话有效，用于存储临时上下文和对话历史。

特点

  • 会话结束后自动清除
  • 存储当前对话上下文
  • 不需要手动管理

自动记录内容

# Claude 自动记录的内容：
- 对话历史
- 当前任务上下文
- 临时决策
- 中间结果

手动保存会话记忆

# 保存临时信息
> /memory save --type session "当前任务: 重构用户模块"

# 查看会话记忆
> /memory list --type session

全局记忆

概述

全局记忆存储通用知识和最佳实践，适用于所有项目。

存储位置

~/.claude/memory/global.json

使用场景

# 保存通用知识
> /memory save --type global "React 最佳实践: 使用函数组件和 Hooks"

# 保存安全知识
> /memory save --type global "安全: 永远不要在前端存储敏感信息"

# 保存性能优化知识
> /memory save --type global "性能: 使用 React.memo 避免不必要的重渲染"

# 保存常用解决方案
> /memory save --type global "解决方案: CORS 问题需要在服务端配置"

记忆结构

{
  "id": "global-001",
  "type": "global",
  "created": "2026-04-04T10:00:00Z",
  "updated": "2026-04-04T10:30:00Z",
  "content": {
    "best_practices": {
      "react": "使用函数组件和 Hooks",
      "security": "不在前端存储敏感信息"
    },
    "solutions": {
      "cors": "服务端配置 Access-Control-Allow-Origin"
    }
  },
  "tags": ["knowledge", "best-practices"],
  "priority": 1
}

记忆优先级

优先级规则

当多个记忆类型包含冲突信息时，按以下优先级处理：

项目记忆 (最高) > 用户记忆 > 全局记忆 (最低)

示例场景

# 全局记忆
"缩进: 4 空格"

# 用户记忆
"缩进: 2 空格"

# 项目记忆
"缩进: Tab"

# 最终使用: Tab（项目记忆优先级最高）

优先级设置

# 设置记忆优先级
> /memory save --priority 10 "重要配置项"

# 高优先级记忆会被优先使用
# 默认优先级: 项目=10, 用户=5, 全局=1

记忆生命周期

创建

# 手动创建
> /memory save "内容"

# 自动创建（Claude 自动保存重要信息）
# Claude 会自动识别并保存重要决策

更新

# 更新记忆
> /memory update [id] "新内容"

# 追加内容
> /memory append [id] "追加的内容"

过期

# 设置过期时间
> /memory save --expires 30d "临时配置"

# 会话记忆自动过期
# 其他类型记忆需要手动删除或设置过期

删除

# 删除单个记忆
> /memory delete [id]

# 清除类型记忆
> /memory clear --type project

# 清除所有记忆
> /memory clear --all
