代理编排

来源说明：本文内容基于 Claude Code 官方文档和最佳实践整理。

可靠性评级：A 官方文档 + 实践验证

编排概述

代理编排是指协调多个子代理协同工作的过程，包括任务分配、执行顺序、结果合并等。

编排模式

  • 顺序执行：按顺序依次执行
  • 并行执行：多个代理同时执行
  • 条件执行：根据条件决定执行路径
  • 循环执行：重复执行直到满足条件

顺序编排

定义工作流

{
  "name": "code-pipeline",
  "type": "sequential",
  "steps": [
    {
      "name": "analyze",
      "agent": "code-analyzer",
      "inputs": {
        "file": "${input.file}"
      }
    },
    {
      "name": "review",
      "agent": "code-reviewer",
      "inputs": {
        "file": "${input.file}",
        "analysis": "${steps.analyze.result}"
      }
    },
    {
      "name": "fix",
      "agent": "code-fixer",
      "inputs": {
        "file": "${input.file}",
        "issues": "${steps.review.issues}"
      }
    }
  ]
}

执行流程

输入文件
    ↓
代码分析代理 → 分析结果
    ↓
代码审查代理 → 审查报告
    ↓
代码修复代理 → 修复后代码
    ↓
输出结果

条件编排

条件分支

{
  "name": "smart-review",
  "type": "conditional",
  "conditions": [
    {
      "when": "${input.file.extension == 'js'}",
      "then": {
        "agent": "js-reviewer"
      }
    },
    {
      "when": "${input.file.extension == 'py'}",
      "then": {
        "agent": "python-reviewer"
      }
    },
    {
      "when": "${input.file.extension == 'ts'}",
      "then": {
        "agent": "typescript-reviewer"
      }
    }
  ],
  "default": {
    "agent": "general-reviewer"
  }
}

复杂条件

{
  "conditions": [
    {
      "when": {
        "and": [
          {"file.extension": "js"},
          {"file.size": {"<": 10000}}
        ]
      },
      "then": {
        "agent": "quick-reviewer"
      }
    },
    {
      "when": {
        "or": [
          {"file.extension": "ts"},
          {"file.extension": "tsx"}
        ]
      },
      "then": {
        "agent": "typescript-reviewer"
      }
    }
  ]
}

循环编排

遍历执行

{
  "name": "batch-review",
  "type": "loop",
  "iterator": {
    "source": "${input.files}",
    "variable": "file"
  },
  "body": {
    "agent": "code-reviewer",
    "inputs": {
      "file": "${file}"

  },
  "collect": {
    "strategy": "array",
    "output": "reports"
  }
}

条件循环

{
  "name": "iterative-fix",
  "type": "while",
  "condition": "${issues.length > 0}",
  "maxIterations": 5,
  "body": [
    {
      "agent": "code-fixer",
      "inputs": {
        "issues": "${issues}"
      }
    },
    {
      "agent": "code-reviewer",
      "outputs": {
        "issues": "issues"
      }
    }
  ]
}

组合编排

嵌套工作流

{
  "name": "full-pipeline",
  "type": "sequential",
  "steps": [
    {
      "name": "parallel-review",
      "type": "parallel",
      "branches": [
        {"agent": "security-reviewer"},
        {"agent": "performance-reviewer"}
      ]
    },
    {
      "name": "conditional-fix",
      "type": "conditional",
      "conditions": [
        {
          "when": "${steps.parallel-review.hasIssues}",
          "then": {"agent": "code-fixer"}
        }
      ],
      "default": {"action": "skip"}
    },
    {
      "agent": "report-generator",
      "inputs": {
        "reviewResults": "${steps.parallel-review}",
        "fixResults": "${steps.conditional-fix}"
      }
    }
  ]
}

错误处理

错误策略

{
  "errorHandling": {
    "strategy": "continue",
    "fallback": {
      "agent": "fallback-agent"
    },
    "retry": {
      "maxAttempts": 3,
      "delay": 1000,
      "backoff": "exponential"
    },
    "timeout": {
      "duration": 60000,
      "action": "cancel"
    }
  }
}

错误传播

{
  "errorPropagation": {
    "mode": "bubble",

        "errorType": "timeout",

      },
      {
        "errorType": "validation",
        "action": "skip"
      },
      {
        "errorType": "*",
        "action": "fail"
      }
    ]
  }
}
