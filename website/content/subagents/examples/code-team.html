代码团队代理案例

来源说明：本文展示一个完整的代码团队代理系统实现。

可靠性评级：B 社区实践

团队概述

代码团队代理是一个多代理协作系统，模拟真实开发团队的工作流程，包括架构师、开发者、审查者等角色。

团队成员

代码团队
├── 架构师代理 (Architect)
│   └── 负责架构设计和决策
├── 开发者代理 (Developer)
│   └── 负责代码实现
├── 审查者代理 (Reviewer)
│   └── 负责代码审查
└── 测试者代理 (Tester)
    └── 负责测试验证

架构师代理

配置

{
  "name": "architect",
  "description": "架构师代理",
  "role": "设计系统架构和技术方案",
  
  "capabilities": [
    "architecture-design",
    "tech-selection",
    "code-structure"
  ],
  
  "tools": ["Read", "Write", "SearchCodebase"],
  
  "prompt": "prompts/architect.md"
}

提示词

# prompts/architect.md

你是一位资深架构师。你的职责是：

1. 分析需求，设计系统架构
2. 选择合适的技术栈
3. 定义代码结构和规范
4. 制定技术决策

团队工作流

工作流配置

{
  "name": "feature-development",
  "type": "sequential",
  "steps": [
    {
      "name": "design",
      "agent": "architect",
      "inputs": {
        "task": "${input.requirement}"
      },
      "outputs": {
        "architecture": "architecture"
      }
    },
    {
      "name": "develop",
      "agent": "developer",
      "inputs": {
        "task": "${input.requirement}",
        "architecture": "${steps.design.architecture}"
      },
      "outputs": {
        "code": "code"
      }
    },
    {
      "name": "review",
      "agent": "reviewer",
      "inputs": {
        "code": "${steps.develop.code}"
      },
      "outputs": {
        "reviewResult": "reviewResult"
      }
    },
    {
      "name": "revise",
      "type": "conditional",
      "condition": "${steps.review.reviewResult.needsRevision}",
      "then": {
        "agent": "developer",
        "inputs": {
          "task": "根据审查意见修改代码",
          "issues": "${steps.review.reviewResult.issues}"
        }
      }
    }
  ]
}

使用示例

启动团队任务

# 启动功能开发
> /team run feature-development --requirement "实现用户登录功能"

# 查看团队状态
> /team status

# 查看各代理输出
> /team logs

执行流程

1. 架构师代理设计架构
   ↓ 输出：架构设计文档
   
2. 开发者代理实现代码
   ↓ 输出：实现代码
   
3. 审查者代理审查代码
   ↓ 输出：审查报告
   
4. 如果需要修改
   ↓ 开发者代理修改代码
   
5. 最终输出
   → 完成的功能代码
