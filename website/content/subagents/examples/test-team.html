测试团队代理案例

来源说明：本文展示一个完整的测试团队代理系统实现。

可靠性评级：B 社区实践

团队概述

测试团队代理是一个专门负责测试的多代理系统，包括测试规划、测试编写、测试执行等角色。

团队成员

测试团队
├── 测试规划代理 (Test Planner)
│   └── 制定测试策略和计划
├── 测试编写代理 (Test Writer)
│   └── 编写测试用例和代码
├── 测试执行代理 (Test Runner)
│   └── 执行测试并收集结果
└── 测试报告代理 (Test Reporter)
    └── 生成测试报告和分析

团队工作流

工作流配置

{
  "name": "test-pipeline",
  "type": "sequential",
  "steps": [
    {
      "name": "plan",
      "agent": "test-planner",
      "inputs": {
        "code": "${input.code}",
        "requirements": "${input.requirements}"
      },
      "outputs": {
        "testPlan": "testPlan"
      }
    },
    {
      "name": "write",
      "agent": "test-writer",
      "inputs": {
        "testPlan": "${steps.plan.testPlan}",
        "code": "${input.code}"
      },
      "outputs": {
        "testFiles": "testFiles"
      }
    },
    {
      "name": "run",
      "agent": "test-runner",
      "inputs": {
        "testFiles": "${steps.write.testFiles}"
      },
      "outputs": {
        "results": "results"
      }
    },
    {
      "name": "report",
      "agent": "test-reporter",
      "inputs": {
        "results": "${steps.run.results}"
      },
      "outputs": {
        "report": "report"
      }
    }
  ]
}

使用示例

启动测试流程

# 为代码生成测试
> /team run test-pipeline --code src/utils.js

# 查看测试计划
> /team output plan

# 查看测试结果
> /team output results

# 查看测试报告
> /team output report
