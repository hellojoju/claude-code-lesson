子代理概念

来源说明：本文内容基于 Claude Code 官方文档和源码分析整理。

可靠性评级：A 官方文档 + 源码验证

什么是子代理

子代理（Subagent）是 Claude Code 中的独立执行单元，可以并行处理特定任务。每个子代理都有自己的上下文和执行环境，可以独立完成分配的任务。

核心特点

  • 独立性：每个子代理独立运行，互不干扰
  • 并行性：多个子代理可以同时执行
  • 专业性：每个子代理专注于特定领域
  • 协作性：子代理之间可以协作完成复杂任务

为什么需要子代理

解决的问题

  • 复杂任务分解：将大任务分解为小任务并行处理
  • 专业分工：不同领域的任务由专门的代理处理
  • 效率提升：并行执行提高整体效率
  • 上下文隔离：避免不同任务的上下文干扰

应用场景

# 单代理模式
用户请求 → Claude 处理 → 返回结果

# 子代理模式
用户请求 → 主代理协调
              ├── 代码审查代理 → 审查结果
              ├── 测试生成代理 → 测试代码
              └── 文档生成代理 → 文档内容
         → 汇总结果 → 返回用户

子代理类型

类型 说明 典型用途
搜索代理 搜索和检索信息 代码搜索、文档查找
代码代理 处理代码相关任务 代码生成、重构
测试代理 处理测试相关任务 测试生成、测试执行
文档代理 处理文档相关任务 文档生成、文档更新
审查代理 审查和检查代码 代码审查、安全检查

基本架构

层级结构

主代理 (Main Agent)
├── 任务分析
├── 子代理调度
├── 结果汇总
└── 响应生成

子代理 (Subagent)
├── 任务接收
├── 独立执行
├── 结果返回
└── 状态报告

通信机制

# 主代理 → 子代理
- 任务分配
- 上下文传递
- 参数配置

# 子代理 → 主代理
- 执行结果
- 进度报告
- 错误信息

基本使用

查看子代理

# 列出所有子代理
> /subagent list

# 查看子代理详情
> /subagent info search-agent

调用子代理

# 直接调用子代理
> /subagent run search-agent "查找所有 TODO"

# 在对话中触发
> 使用搜索代理查找所有 console.log

并行执行

# 并行执行多个子代理
> /subagent parallel \
    search-agent "查找 API 调用" \
    review-agent "审查安全问题" \
    doc-agent "生成 API 文档"

子代理生命周期

状态流转

创建 → 初始化 → 运行 → 完成/失败 → 清理

状态说明

状态 说明
created 已创建，等待初始化
initialized 已初始化，准备执行
running 正在执行任务
completed 任务完成
failed 任务失败
