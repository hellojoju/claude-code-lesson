主代理架构

核心组件

主代理
├── 任务解析器
│   ├── 意图识别
│   ├── 任务分解
│   └── 参数提取
├── 调度器
│   ├── 代理选择
│   ├── 任务分配
│   └── 并行调度
├── 上下文管理器
│   ├── 上下文构建
│   ├── 上下文传递
│   └── 上下文合并
└── 结果处理器
    ├── 结果收集
    ├── 结果合并
    └── 响应生成

工作流程

1. 接收用户请求
2. 分析任务类型和复杂度
3. 决定是否需要子代理
4. 选择合适的子代理
5. 分配任务并传递上下文
6. 监控执行状态
7. 收集和合并结果
8. 生成最终响应

通信机制

消息类型

// 任务分配消息
{
  "type": "task_assignment",
  "taskId": "task-001",
  "agentId": "search-agent",
  "task": {
    "description": "搜索所有 TODO",
    "parameters": {
      "pattern": "TODO",
      "scope": "src/"
    }
  },
  "context": {
    "project": {...},
    "memory": {...}
  }
}

// 结果返回消息
{
  "type": "task_result",
  "taskId": "task-001",
  "agentId": "search-agent",
  "status": "completed",
  "result": {
    "matches": [...],
    "count": 10
  }
}

通信模式

  • 同步模式：主代理等待子代理完成
  • 异步模式：子代理独立运行，定期报告
  • 流式模式：子代理实时返回中间结果

调度策略

调度算法

# 任务分解策略
1. 分析任务复杂度
2. 识别可并行部分
3. 确定依赖关系

# 代理选择策略
1. 匹配任务类型
2. 检查代理可用性
3. 考虑负载均衡
4. 选择最优代理

错误处理

错误类型

错误类型 处理策略
超时错误 重试或降级处理
资源错误 等待资源或切换代理
执行错误 错误恢复或报告失败
通信错误 重连或使用备用通道

容错机制

# 重试策略
maxRetries: 3
retryDelay: 1000ms (指数退避)

# 超时设置
taskTimeout: 60000ms
agentTimeout: 300000ms

# 降级策略
fallbackAgent: "default-agent"
fallbackAction: "report-error"
