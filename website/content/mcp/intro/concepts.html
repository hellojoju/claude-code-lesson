MCP概念详解

可靠性: A 来源: MCP 官方规范 + Claude Code 文档

更新时间: 2025-04-04 作者: Claude Code 教程团队

什么是 MCP

MCP (Model Context Protocol) 是 Anthropic 主导的开放协议，用于在 AI 模型和外部数据源、工具之间建立标准化连接。它被形容为"AI 的 USB-C"，让 Claude Code 能够安全地访问数据库、API、文件系统等外部资源。

核心价值：MCP 让 Claude Code 从一个代码助手变成可以连接整个数字世界的智能代理。

核心优势

标准化

统一的接口规范，一次集成到处可用。

安全性

内置权限控制和审计机制，保护敏感数据。

可扩展

支持自定义服务器，满足特定需求。

互操作性

跨平台、跨语言支持，生态丰富。

核心组件

组件 说明 作用
MCP 客户端 Claude Code 内置 管理服务器连接、调用工具
MCP 服务器 独立进程 连接外部资源、提供工具
传输层 通信协议 stdio、HTTP、WebSocket
资源层 外部系统 数据库、API、文件系统

传输类型

STDIO

特点：本地进程通信，单客户端

适用：本地工具、命令行程序

SSE

特点：远程 HTTP，支持多客户端

适用：云服务、远程 API

Streamable HTTP

特点：支持流式响应

适用：Serverless、自动扩展

核心功能

资源访问

MCP 提供统一的资源访问接口：

# 通过 @ 语法访问 MCP 资源
> 查看 @github:repos/owner/repo/issues
> 读取 @notion:page-id 的内容
> 搜索 @slack:channel-name 的消息

工具调用

MCP 服务器可以提供各种工具：

工具类型 功能 示例
查询工具 执行数据库查询 postgres.query
操作工具 创建、更新、删除数据 github.create_issue
搜索工具 搜索和检索信息 brave.search
分析工具 分析数据结果 puppeteer.screenshot

提示词模板

MCP 服务器可以提供预定义的提示词模板：

# 使用 MCP 提供的提示词模板
> 使用 @github:templates/pr-review 审查这个 PR
> 使用 @postgres:templates/schema-analysis 分析数据库结构

服务器 功能 需要 API Key

安装配置

一键安装脚本（macOS/Linux）

bash <<'EOF'
echo "正在安装 Claude MCP 服务器..."

# 顺序思考
claude mcp add sequential-thinking -s user \
  -- npx -y @modelcontextprotocol/server-sequential-thinking || true

# 文件系统访问
claude mcp add filesystem -s user \
  -- npx -y @modelcontextprotocol/server-filesystem \
     ~/Documents ~/Desktop ~/Downloads ~/Projects || true

# Playwright（多浏览器自动化）
claude mcp add playwright -s user \
  -- npx -y @playwright/mcp || true

# Web Fetching
claude mcp add fetch -s user \
  -- npx -y @kazuph/mcp-fetch || true

echo "✅ MCP 注册完成。"
claude mcp list
EOF

手动配置

// ~/.claude/settings.json 或 .claude/settings.json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/project"],
      "env": {"ALLOWED_PATHS": "/project"}
    },
    "github": {
      "type": "http",
      "url": "https://api.githubcopilot.com/mcp",
      "headers": {"Authorization": "Bearer ${GITHUB_TOKEN}"}
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {"DATABASE_URL": "${POSTGRES_URL}"}
    }
  }
}

安装范围说明

参数 含义 配置文件位置
-s user 全局安装 ~/.claude.json
-s local 项目内安装 .claude.json

基本使用

管理 MCP 服务器

# 列出所有服务器
claude mcp list

# 添加服务器
claude mcp add server-name -s user -- npx -y package-name

# 删除服务器
claude mcp remove server-name

# 启用/禁用服务器
@server-name enable
@server-name disable

使用 MCP 资源

# 在对话中引用 MCP 资源
> 查看 @github:repos/owner/repo/issues
> 读取 @filesystem:path/to/file.txt
> 搜索 @brave-search:query

监控 MCP 使用

# 查看上下文使用情况
> /context

# 查看 MCP 工具占用的 token
# 输出示例：
# MCP 服务器 token 使用：
# - github: 5,000 tokens
# - filesystem: 2,000 tokens
# - postgres: 1,500 tokens

调试 MCP

# 启用调试模式
claude --mcp-debug

# 查看服务器日志
cat ~/.claude/logs/mcp.log

安全机制

权限控制

{
  "permissions": {
    "mcpServers": {
      "postgres": {
        "allow": ["query", "insert", "update"],
        "deny": ["delete", "drop"],
        "tables": ["users", "orders"]
      }
    }
  }
}

敏感信息保护

安全提示：

  • 不要在配置文件中硬编码 API Key
  • 使用环境变量存储敏感信息
  • 定期审查 MCP 服务器的权限
  • 只启用必要的服务器

审计日志

{
  "audit": {
    "enabled": true,
    "logFile": "~/.claude/mcp-audit.log",
    "level": "verbose"
  }
}

最佳实践

选择性启用

只启用项目需要的 MCP 服务器，减少 token 消耗。

环境变量

使用环境变量存储 API Key，不要硬编码。

定期审查

定期检查 MCP 服务器的权限和日志。

版本锁定

锁定 MCP 服务器版本，避免意外更新。

常见问题

MCP 显示连接但无响应？

检查路径配置和通信状态：

# 检查服务器状态
claude mcp list

# 设置超时
export MCP_TIMEOUT=10000

Token 消耗过多？

禁用未使用的 MCP 服务器：

# 禁用服务器
@server-name disable

# 或删除不需要的服务器
claude mcp remove server-name
