调试命令详解

来源说明：本文内容基于 Claude Code 官方文档和实际使用案例整理。

可靠性评级：A 官方文档 + 实践验证

调试功能概述

Claude Code 提供了强大的调试命令，帮助开发者快速定位和解决问题。

调试命令列表

命令 功能 使用场景
/debug 启动调试会话 调试运行时问题
/analyze 分析代码问题 静态代码分析
/trace 追踪执行流程 理解代码逻辑
/profile 性能分析 性能优化
/log 日志管理 查看和过滤日志

/debug - 调试会话

启动调试

# 调试当前文件
> /debug src/index.js

# 调试测试文件
> /debug tests/app.test.js

# 调试特定函数
> /debug src/utils.js --function formatDate

断点管理

# 设置断点
> /debug --breakpoint src/index.js:25

# 条件断点
> /debug --breakpoint src/index.js:30 --condition "x > 10"

# 删除断点
> /debug --remove-breakpoint src/index.js:25

# 列出所有断点
> /debug --list-breakpoints

调试控制

# 调试会话中的控制命令
continue (c)    - 继续执行
step (s)        - 单步执行（进入函数）
next (n)        - 单步执行（不进入函数）
finish (f)      - 执行到函数返回
breakpoint (b)  - 设置断点
watch (w)       - 添加监视表达式
print (p)       - 打印变量值
backtrace (bt)  - 显示调用栈
quit (q)        - 退出调试

实际案例

# 调试 API 请求问题
> /debug src/api.js --breakpoint src/api.js:50
> 在断点处检查 request 对象的值

# 调试异步代码
> /debug src/async.js --async
> 追踪 Promise 链的执行

# 调试内存泄漏
> /debug --heap-snapshot
> 分析内存使用情况

/analyze - 代码分析

基本分析

# 分析单个文件
> /analyze src/index.js

# 分析整个项目
> /analyze src/

# 分析特定类型的问题
> /analyze src/ --type security

分析类型

# 安全分析
> /analyze --type security

# 性能分析
> /analyze --type performance

# 代码质量
> /analyze --type quality

# 复杂度分析
> /analyze --type complexity

# 依赖分析
> /analyze --type dependencies

/trace - 执行追踪

基本追踪

# 追踪函数调用
> /trace src/main.js --function processOrder

# 追踪模块加载
> /trace --modules

# 追踪 API 调用
> /trace --api-calls

追踪选项

# 设置追踪深度
> /trace src/main.js --depth 5

# 追踪特定变量
> /trace src/main.js --watch user

# 追踪异步操作
> /trace --async

# 输出追踪结果
> /trace src/main.js --output trace.log

/profile - 性能分析

性能剖析

# 启动性能分析
> /profile start

# 运行代码
> /run npm test

# 停止并查看报告
> /profile stop --report

/log - 日志管理

查看日志

# 查看最近日志
> /log --tail 100

# 实时查看日志
> /log --follow

# 过滤日志
> /log --filter "error"

日志级别

# 按级别过滤
> /log --level error
> /log --level warn,error
> /log --level debug

# 设置日志级别
> /log --set-level debug

日志分析

# 统计错误
> /log --stats

# 搜索日志
> /log --search "failed"

# 导出日志
> /log --export logs.json

实际调试案例

案例一：调试 API 错误

# 步骤 1: 分析错误日志
> /log --filter "API Error" --context 5

# 步骤 2: 追踪 API 调用
> /trace src/api.js --function fetchData

# 步骤 3: 设置断点调试
> /debug src/api.js --breakpoint src/api.js:45

# 步骤 4: 分析问题
> 检查请求参数和响应

案例二：性能优化

# 步骤 1: 性能分析
> /profile start
> /run npm run build
> /profile stop --report

# 步骤 2: 找出瓶颈
> /analyze --type performance

# 步骤 3: 追踪慢函数
> /trace src/slow-function.js

# 步骤 4: 优化并验证
> 优化代码后重新运行分析

案例三：内存泄漏

# 步骤 1: 堆快照
> /debug --heap-snapshot baseline.heapsnapshot

# 步骤 2: 运行一段时间
> /run npm start --background

# 步骤 3: 再次快照
> /debug --heap-snapshot after.heapsnapshot

# 步骤 4: 比较分析
> /debug --compare-heap baseline.heapsnapshot after.heapsnapshot
