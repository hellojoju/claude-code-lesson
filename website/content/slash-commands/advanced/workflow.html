工作流命令详解

来源说明：本文内容基于 Claude Code 官方文档和实际使用案例整理。

可靠性评级：A 官方文档 + 实践验证

工作流命令概述

工作流命令帮助开发者自动化常见的开发流程，提高工作效率。

工作流命令列表

命令 功能
/workflow 管理工作流
/review 代码审查
/refactor 代码重构
/test 测试管理
/deploy 部署操作

/workflow - 工作流管理

创建工作流

# 创建新工作流
> /workflow create feature-development

# 定义工作流步骤
> /workflow add-step "运行测试" --command "npm test"
> /workflow add-step "代码检查" --command "npm run lint"
> /workflow add-step "构建" --command "npm run build"

执行工作流

# 运行工作流
> /workflow run feature-development

# 运行特定步骤
> /workflow run feature-development --step "运行测试"

# 并行执行
> /workflow run feature-development --parallel

工作流配置文件

# .claude/workflows/feature-development.yaml
name: feature-development
description: 功能开发工作流

steps:
  - name: 安装依赖
    command: npm install
    
  - name: 运行测试
    command: npm test
    
  - name: 代码检查
    command: npm run lint
    
  - name: 构建
    command: npm run build
    
  - name: 生成文档
    command: npm run docs

/review - 代码审查

基本审查

# 审查当前更改
> /review

# 审查特定文件
> /review src/index.js

# 审查 Git diff
> /review --git-diff

审查选项

# 指定审查重点
> /review --focus security
> /review --focus performance
> /review --focus style

# 审查范围
> /review --scope function
> /review --scope module
> /review --scope project

# 输出格式
> /review --format markdown
> /review --format json

审查报告示例

# 代码审查报告

## 概述
- 审查文件: src/index.js
- 发现问题: 5 个
- 风险等级: 中

## 问题列表

### 🔴 高优先级
1. **第 25 行: SQL 注入风险**
   ```javascript
   const query = `SELECT * FROM users WHERE id = ${userId}`;
   ```
   建议: 使用参数化查询

### 🟡 中优先级
2. **第 42 行: 未处理的 Promise**
   ```javascript
   fetchData().then(data => process(data));
   ```
   建议: 添加 .catch() 处理错误

### 🟢 低优先级
3. **第 15 行: 未使用的变量**
   ```javascript
   const temp = getTemp();
   ```
   建议: 移除未使用的变量

/refactor - 代码重构

基本重构

# 重构文件
> /refactor src/index.js

# 重构函数
> /refactor src/utils.js --function formatDate

# 重构类
> /refactor src/User.js --class User

重构类型

# 提取函数
> /refactor --extract-function --name validateEmail

# 提取变量
> /refactor --extract-variable --name MAX_RETRY

# 内联函数
> /refactor --inline-function

# 重命名
> /refactor --rename oldName newName

# 移动文件
> /refactor --move src/old.js src/new.js

重构案例

# 案例 1: 提取重复代码
> /search "重复的代码块" --type js
> /refactor --extract-function --name commonLogic

# 案例 2: 简化条件
> /refactor src/index.js --simplify-conditions

# 案例 3: 转换为 async/await
> /refactor src/api.js --convert-async

/test - 测试管理

运行测试

# 运行所有测试
> /test

# 运行特定测试文件
> /test user.test.js

# 运行匹配的测试
> /test --grep "should create"

# 运行覆盖率
> /test --coverage

生成测试

# 为文件生成测试
> /test --generate src/utils.js

# 生成特定类型的测试
> /test --generate src/api.js --type integration

# 生成快照测试
> /test --snapshot src/components/Button.jsx

/deploy - 部署操作

部署命令

# 部署到生产环境
> /deploy production

# 部署到测试环境
> /deploy staging

# 部署特定分支
> /deploy --branch feature/new-ui

# 部署特定版本
> /deploy --version 1.2.3

部署前检查

# 运行部署前检查
> /deploy --check

# 检查项包括:
# - 测试是否通过
# - 代码是否已提交
# - 依赖是否更新
# - 配置是否正确

回滚操作

# 查看部署历史
> /deploy --history

# 回滚到上一版本
> /deploy --rollback

# 回滚到特定版本
> /deploy --rollback --version 1.2.0

实用工作流示例

功能开发工作流

# feature-flow.yaml
name: feature-flow
steps:
  - name: 创建分支
    command: git checkout -b feature/${feature-name}
    
  - name: 开发功能
    type: interactive
    
  - name: 编写测试
    command: /test --generate
    
  - name: 运行测试
    command: npm test
    
  - name: 代码审查
    command: /review
    
  - name: 提交代码
    command: git add . && git commit -m "feat: ${message}"

发布工作流

# release-flow.yaml
name: release-flow
steps:
  - name: 检查测试
    command: npm test
    
  - name: 检查代码质量
    command: npm run lint
    
  - name: 构建项目
    command: npm run build
    
  - name: 更新版本
    command: npm version ${version-type}
    
  - name: 生成变更日志
    command: npm run changelog
    
  - name: 发布
    command: npm publish
