文件操作命令详解

来源说明：本文内容基于 Claude Code 官方文档和实际使用案例整理。

可靠性评级：A 官方文档 + 实践验证

/read - 读取文件

基本用法

# 读取单个文件
> /read src/index.js

# 读取多个文件
> /read src/index.js src/app.js

# 读取目录下所有文件
> /read src/

高级选项

# 显示行号
> /read src/index.js --line-numbers

# 指定行范围
> /read src/index.js --lines 10-50

# 显示语法高亮
> /read src/index.js --highlight

# 显示文件信息
> /read src/index.js --info

实际案例

# 读取后分析
> /read package.json
> 分析这个项目的依赖，找出可能过时的包

# 读取配置文件
> /read .env.example
> 根据这个模板创建 .env 文件

# 读取测试文件
> /read tests/user.test.js
> 为这些测试生成对应的实现代码

/write - 写入文件

基本用法

# 创建新文件
> /write newfile.js "console.log('Hello');"

# 覆盖现有文件
> /write config.json '{"name": "my-project"}'

# 写入多行内容
> /write readme.md <

安全选项

# 追加模式
> /write log.txt "新日志条目" --append

# 仅当文件不存在时创建
> /write config.json "内容" --create-only

# 备份原文件
> /write important.js "新内容" --backup

实际案例

# 创建组件文件
> 创建一个 React 组件 Button，保存到 src/components/Button.jsx
> /write src/components/Button.jsx [组件代码]

# 更新配置
> /read package.json
> 添加 eslint 和 prettier 依赖
> /write package.json [更新后的内容]

/edit - 编辑文件

基本用法

# 交互式编辑
> /edit src/index.js

# 替换内容
> /edit src/index.js --replace "oldFunction" --with "newFunction"

# 在指定行插入
> /edit src/index.js --insert-at 10 "// 新注释"

高级编辑

# 正则替换
> /edit src/index.js --regex "var\s+(\w+)" --with "const $1"

# 多文件编辑
> /edit "*.js" --replace "var" --with "const"

# 条件编辑
> /edit src/index.js --if-contains "TODO" --append "\n// 已完成"

实际案例

# 重命名变量
> /edit src/index.js --replace "oldName" --with "newName"

# 添加导入语句
> /edit src/index.js --insert-at 1 "import React from 'react';"

# 批量修改
> 把所有 console.log 改成 console.debug
> /edit "src/**/*.js" --regex "console\.log\((.*?)\)" --with "console.debug($1)"

/search - 搜索内容

基本搜索

# 搜索文本
> /search "function"

# 搜索正则表达式
> /search "/function\s+\w+/g"

# 搜索整个词
> /search "test" --whole-word

过滤选项

# 按文件类型
> /search "import" --type js

# 按目录
> /search "TODO" --dir src

# 排除目录
> /search "password" --exclude node_modules,dist

# 大小写敏感
> /search "React" --case-sensitive

输出格式

# 显示上下文
> /search "error" --context 3

# 只显示文件名
> /search "deprecated" --files-only

# 显示行号
> /search "function" --line-numbers

# JSON 格式输出
> /search "TODO" --format json

实际案例

# 查找所有 TODO
> /search "TODO|FIXME" --type js --context 2

# 查找未使用的导入
> /search "^import.*from" --type js
> 分析哪些导入没有被使用

# 安全审计
> /search "password|secret|key" --exclude "*.test.js"
> 检查是否有敏感信息泄露

/glob - 文件模式匹配

基本用法

# 查找所有 JS 文件
> /glob "*.js"

# 查找多级目录
> /glob "src/**/*.js"

# 多种模式
> /glob "*.{js,ts,jsx,tsx}"

# 排除模式
> /glob "src/**/*.js" --exclude "**/*.test.js"

常用模式

*           匹配任意字符（不含路径分隔符）
**          匹配任意字符（含路径分隔符）
?           匹配单个字符
[abc]       匹配指定字符
{a,b,c}     匹配任意一个模式
!pattern    排除模式

实际案例

# 查找所有组件
> /glob "src/components/**/*.{js,jsx,ts,tsx}"

# 查找配置文件
> /glob "*.{json,yaml,yml,toml}"

# 查找测试文件
> /glob "**/*.test.{js,ts}"

# 统计代码行数
> /glob "src/**/*.js" | 统计总行数

/copy 和 /move - 复制和移动

复制文件

# 复制单个文件
> /copy src/index.js backup/index.js

# 复制目录
> /copy src/ backup/src/

# 复制多个文件
> /copy file1.js file2.js destination/

移动文件

# 移动文件
> /move old/path.js new/path.js

# 重命名
> /move component.js Component.jsx

# 移动目录
> /move old-dir/ new-dir/

/delete - 删除文件

警告：删除操作不可逆，请谨慎使用！

基本用法

# 删除文件（需要确认）
> /delete temp.txt

# 删除多个文件
> /delete file1.js file2.js

# 删除目录
> /delete dist/

安全选项

# 仅删除空目录
> /delete empty-dir/ --only-empty

# 交互式确认
> /delete "*.log" --interactive

# 模拟删除（不实际执行）
> /delete temp/ --dry-run
