代码质量类命令

代码复杂度分析

# ~/.claude/commands/complexity.md

---
name: complexity
description: 分析代码复杂度
---

分析以下代码的复杂度：

{{selection}}

## 分析维度

1. **圈复杂度**
   - 计算条件分支数量
   - 识别嵌套层级
   - 评估可读性

2. **认知复杂度**
   - 评估理解难度
   - 识别心理负担点
   - 建议简化方案

3. **代码行数**
   - 函数长度
   - 文件长度
   - 注释比例

## 输出格式

```
复杂度报告
==========

圈复杂度: X (建议 < 10)
认知复杂度: X (建议 < 15)
函数行数: X (建议 < 50)

建议:
- [具体建议列表]
```

代码规范检查

# ~/.claude/commands/lint-check.md

---
name: lint-check
description: 自定义代码规范检查
---

检查以下代码是否符合规范：

{{selection}}

## 检查规则

### 命名规范
- 变量使用 camelCase
- 常量使用 UPPER_SNAKE_CASE
- 类名使用 PascalCase
- 文件名使用 kebab-case

### 格式规范
- 缩进使用 2 空格
- 字符串使用单引号
- 语句末尾不加分号
- 最大行宽 100 字符

### 最佳实践
- 使用 const/let 替代 var
- 箭头函数优先
- 解构赋值优先
- 模板字符串优先

## 输出

列出所有不符合规范的代码行，并提供修复建议。

单元测试生成

# ~/.claude/commands/gen-test.md

---
name: gen-test
description: 生成单元测试
parameters:
  - name: framework
    default: jest
---

为以下代码生成单元测试：

{{selection}}

测试框架: {{framework}}

## 测试要求

1. **测试覆盖**
   - 正常情况测试
   - 边界情况测试
   - 异常情况测试

2. **测试结构**
   ```javascript
   describe('函数名', () => {
     describe('场景描述', () => {
       it('应该...', () => {
         // arrange
         // act
         // assert
       });
     });
   });
   ```

3. **测试原则**
   - 每个测试只验证一个行为
   - 使用描述性的测试名称
   - 遵循 AAA 模式

## 输出

生成完整的测试文件，包含所有必要的导入和模拟。

   - 添加函数注释

2. 替换原代码

代码现代化

# ~/.claude/commands/modernize.md

---
name: modernize
description: 将代码升级为现代语法
---

将以下代码升级为现代 JavaScript/TypeScript 语法：

{{selection}}

## 转换规则

1. **变量声明**
   - var → const/let
   - 根据是否重新赋值选择

2. **函数语法**
   - function → 箭头函数（适当场景）
   - 添加类型注解（TypeScript）

3. **异步处理**
   - callback → Promise
   - .then() → async/await

4. **模块系统**
   - require → import
   - module.exports → export

5. **其他**
   - 字符串拼接 → 模板字符串
   - for 循环 → for...of/forEach
   - arguments → 剩余参数

## 输出

显示转换后的代码，并说明每个改动。

Git 相关命令

提交信息生成

# ~/.claude/commands/commit-msg.md

---
name: commit-msg
description: 生成规范的提交信息
---

根据当前更改生成提交信息。

## 分析内容

1. 查看 git diff
2. 分析更改类型
3. 生成提交信息

## 提交信息格式

```
(): 





```

### 类型
- feat: 新功能
- fix: 修复 bug
- docs: 文档更改
- style: 代码格式
- refactor: 重构
- test: 测试相关
- chore: 构建/工具

### 示例

```
feat(auth): 添加用户登录功能

- 实现 JWT 认证
- 添加登录表单验证
- 集成第三方登录

Closes #123
```

分支管理

# ~/.claude/commands/branch.md

---
name: branch
description: 智能分支管理
parameters:
  - name: action
    options: [create, switch, delete, list]
  - name: branchName
---

执行分支操作: {{action}}

## 操作说明

### create
创建新分支，命名规范：
- feature/xxx: 新功能
- fix/xxx: bug 修复
- refactor/xxx: 重构
- docs/xxx: 文档更新

### switch
切换分支，自动保存当前更改

### delete
删除已合并的分支

### list
列出所有分支及其状态

依赖分析

# ~/.claude/commands/deps.md

---
name: deps
description: 分析项目依赖
---

分析 package.json 中的依赖。

## 分析内容

1. **依赖概览**
   - 总数统计
   - 类型分布

2. **安全检查**
   - 已知漏洞
   - 风险等级

3. **更新检查**
   - 过时依赖
   - 最新版本
   - 破坏性变更

4. **优化建议**
   - 未使用的依赖
   - 可替换的依赖
   - 重复功能依赖

## 输出格式

```
依赖分析报告
============

📦 总依赖: 50
  生产依赖: 30
  开发依赖: 20

🔒 安全问题: 2
  🔴 高危: 1
  🟡 中危: 1

📅 过时依赖: 5
  lodash: 4.0.0 → 4.17.21
  ...

💡 优化建议:
  - 移除未使用的依赖: unused-package
  - 替换 moment.js 为 dayjs
```
