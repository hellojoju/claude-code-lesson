创建自定义命令

来源说明：本文内容基于 Claude Code 官方文档和社区最佳实践整理。

可靠性评级：A 官方文档 + 社区验证

自定义命令概述

自定义命令允许你创建专属于你的工作流的命令，提高开发效率。

命令存放位置

~/.claude/commands/          # 用户级命令（全局可用）
.project/commands/           # 项目级命令（仅当前项目）

命令文件格式

自定义命令使用 Markdown 格式，文件名即为命令名：

~/.claude/commands/review.md  → /review
~/.claude/commands/test.md    → /test

创建基础命令

简单命令示例

# ~/.claude/commands/explain.md

请详细解释选中的代码，包括：
1. 代码的功能和目的
2. 使用的设计模式
3. 潜在的问题或改进建议
4. 相关的最佳实践

使用命令

# 使用自定义命令
> /explain src/index.js

# 或在文件中选中代码后使用
> /read src/index.js --lines 10-20
> /explain

带参数的命令

# ~/.claude/commands/translate.md

将以下代码翻译为 {{language}} 语言。
保持相同的功能和逻辑。
使用目标语言的最佳实践和惯用写法。

代码：
{{selection}}

使用带参数的命令

# 指定参数
> /translate --language Python

# 或简写
> /translate Python

命令变量

内置变量

变量 说明
{{selection}} 当前选中的代码或文本
{{file}} 当前文件路径
{{filename}} 当前文件名
{{directory}} 当前目录路径
{{git_branch}} 当前 Git 分支名
{{date}} 当前日期
{{time}} 当前时间

自定义变量

# ~/.claude/commands/create-component.md

创建一个 {{framework}} 组件，组件名为 {{name}}。

要求：
- 使用 TypeScript
- 包含样式文件
- 包含测试文件
- 遵循 {{style_guide}} 规范

使用自定义变量

# 交互式输入
> /create-component

# 直接指定
> /create-component --framework React --name Button --style-guide airbnb

高级功能

条件逻辑

# ~/.claude/commands/smart-review.md

{{#if selection}}
请审查选中的代码：
{{selection}}
{{else}}
请审查当前文件：{{file}}
{{/if}}

审查重点：
- 代码质量
- 安全问题
- 性能考虑

循环处理

# ~/.claude/commands/batch-process.md

批量处理以下文件：
{{#each files}}
文件 {{@index}}: {{this}}
{{/each}}

对每个文件执行：
1. 检查代码规范
2. 添加缺失的类型注释
3. 优化导入语句

命令组合

# ~/.claude/commands/full-review.md

执行完整的代码审查流程：

## 步骤 1: 运行测试
请运行: npm test

## 步骤 2: 代码检查
请运行: npm run lint

## 步骤 3: 安全审计
请运行: npm audit

## 步骤 4: 代码审查
审查文件：{{file}}

## 步骤 5: 生成报告
总结所有发现的问题并提供修复建议。

命令模板

代码审查模板

# ~/.claude/commands/review.md

---
name: review
description: 代码审查命令
author: your-name
version: 1.0
---

# 代码审查

请对以下代码进行全面审查：

{{#if selection}}
```
{{selection}}
```
{{else}}
文件：{{file}}
{{/if}}

## 审查清单

### 代码质量
- [ ] 代码是否清晰易读
- [ ] 命名是否规范
- [ ] 是否有重复代码
- [ ] 函数是否过长

### 安全性
- [ ] 是否有 SQL 注入风险
- [ ] 是否有 XSS 风险
- [ ] 敏感数据是否安全处理

### 性能
- [ ] 是否有不必要的循环
- [ ] 是否有内存泄漏风险
- [ ] 异步操作是否正确处理

### 最佳实践
- [ ] 是否遵循项目规范
- [ ] 是否有足够的注释
- [ ] 错误处理是否完善

## 输出格式

请按以下格式输出审查结果：

### 🔴 高优先级问题
[列出需要立即修复的问题]

### 🟡 中优先级问题
[列出应该修复的问题]

### 🟢 低优先级建议
[列出可以改进的建议]

### ✅ 优点
[列出代码中做得好的地方]

文档生成模板

# ~/.claude/commands/document.md

---
name: document
description: 生成代码文档
---

为以下代码生成文档：

{{selection}}

## 文档要求

1. **函数描述**
   - 功能说明
   - 参数说明（类型、用途、默认值）
   - 返回值说明
   - 异常说明

2. **使用示例**
   - 基本用法
   - 高级用法
   - 边界情况

3. **注意事项**
   - 使用限制
   - 性能考虑
   - 相关依赖

## 输出格式

使用 JSDoc 格式生成文档注释。

命令配置

命令元数据

# ~/.claude/commands/my-command.md

---
name: my-command
description: 命令描述
author: 作者名
version: 1.0.0
category: development
tags:
  - code
  - review
aliases:
  - mc
  - mycmd
parameters:
  - name: language
    description: 目标语言
    type: string
    default: JavaScript
    required: true
  - name: style
    description: 代码风格
    type: string
    default: standard
    options:
      - standard
      - airbnb
      - google
---

测试自定义命令

验证命令语法

# 验证命令文件
> /command validate review.md

# 检查变量引用
> /command check-variables review.md

调试命令

# 启用调试模式
> /review --debug

# 查看命令解析结果
> /review --show-parsed

# 测试变量替换
> /review --test-variables

命令日志

# 查看命令执行日志
> /command logs review

# 查看错误日志
> /command errors

最佳实践

命名规范

  • 使用小写字母和连字符：code-review 而非 codeReview
  • 使用动词开头：create-component、fix-bug
  • 保持简洁：lint 而非 run-linter

文档完善

  • 添加命令描述
  • 说明参数用途
  • 提供使用示例
  • 注明版本和作者

版本管理

# 将自定义命令纳入版本控制
cd ~/.claude/commands
git init
git add .
git commit -m "Initial commands"
