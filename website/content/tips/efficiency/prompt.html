提示词技巧

可靠性: A 来源: Claude Code 官方文档 + 社区经验

编写高效提示词

好的提示词可以显著提高 Claude Code 的响应质量和效率。本节介绍编写高效提示词的技巧。

核心原则

🎯 明确具体

清楚描述你想要什么，避免模糊的表述。

# ❌ 模糊
"帮我优化代码"

# ✅ 具体
"优化 src/utils/format.ts 中的 formatDate 函数，
减少内存分配，提高执行速度"

📋 提供上下文

提供必要的背景信息帮助理解。

# ❌ 缺少上下文
"修复这个 bug"

# ✅ 提供上下文
"修复登录页面的 bug：用户输入正确密码后
仍然显示'密码错误'，检查 src/auth/login.ts
中的验证逻辑"

📝 分步骤

复杂任务分解为多个步骤。

# ✅ 分步骤
"完成以下任务：
1. 分析 src/api/users.ts 的现有代码
2. 添加分页功能
3. 添加排序功能
4. 编写单元测试"

🎭 指定角色

让 Claude 扮演特定角色。

# ✅ 指定角色
"作为安全专家，审查这段代码的安全漏洞"

常用提示词模板

代码生成

创建一个 [组件/函数/类]，实现以下功能：
- 功能点 1
- 功能点 2
- 功能点 3

要求：
- 使用 TypeScript
- 遵循项目编码规范
- 添加必要的注释
- 编写单元测试

代码审查

审查 [文件路径] 的代码，关注：
- 代码质量和可读性
- 潜在的 bug
- 性能问题
- 安全风险
- 最佳实践

提供具体的改进建议。

问题排查

分析以下问题：
[问题描述]

相关代码：
[代码片段]

错误信息：
[错误信息]

请：
1. 分析问题原因
2. 提供解决方案
3. 解释为什么会出现这个问题

高级技巧

链式提示

# 第一步：分析
"分析 src/api/ 目录下的所有 API 端点"

# 第二步：基于分析结果
"根据上面的分析，为每个端点生成 OpenAPI 文档"

# 第三步：继续深化
"为 POST /users 端点添加请求验证"

提供示例

按照以下格式生成代码：

示例输入：
```typescript
interface User {
    id: string;
    name: string;
}
```

示例输出：
```typescript
export interface UserDTO {
    id: string;
    name: string;
    createdAt: Date;
}

export function toUserDTO(user: User): UserDTO {
    return {
        ...user,
        createdAt: new Date()
    };
}
```
