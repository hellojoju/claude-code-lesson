插件概念详解

来源说明：本文内容基于 Claude Code 官方文档和源码分析整理。

可靠性评级：A 官方文档 + 源码验证

什么是插件

插件是 Claude Code 的扩展机制，允许开发者创建独立的功能模块来扩展 Claude Code 的能力。插件可以添加命令、技能、钩子、UI 组件等。

插件 vs 其他扩展机制

扩展机制 复杂度 功能范围 分发方式
插件 高 全面 独立包
技能 中 特定任务 配置文件
钩子 低 事件响应 脚本文件

插件能力

可以做什么

  • 添加命令：注册新的斜杠命令
  • 创建技能：定义可复用的技能模块
  • 注册钩子：响应系统事件
  • 扩展 UI：添加界面元素
  • 集成服务：连接外部服务
  • 修改行为：覆盖默认行为

插件类型

# 按功能分类
- 命令插件：添加新命令
- 集成插件：连接外部服务
- 主题插件：自定义外观
- 工具插件：提供开发工具
- 工作流插件：自动化流程

插件结构

标准目录结构

my-plugin/
├── package.json        # 插件配置
├── src/
│   ├── index.ts       # 入口文件
│   ├── commands/      # 命令定义
│   ├── skills/        # 技能定义
│   ├── hooks/         # 钩子定义
│   └── utils/         # 工具函数
├── tests/             # 测试文件
├── docs/              # 文档
└── README.md          # 说明文档

插件生命周期

生命周期阶段

1. 发现
   - 扫描插件目录
   - 读取 package.json
   
2. 加载
   - 解析配置
   - 验证依赖
   
3. 激活
   - 执行 activate 函数
   - 注册贡献点
   
4. 运行
   - 响应用户操作
   - 执行插件功能
   
5. 停用
   - 执行 deactivate 函数
   - 清理资源

入口文件

// src/index.ts
import { Plugin } from '@anthropic/claude-code-api';

export const plugin: Plugin = {
  // 激活时调用
  async activate(context) {
    console.log('插件已激活');
    
    // 注册命令
    context.registerCommand('my-plugin.hello', async () => {
      console.log('Hello from plugin!');
    });
  },
  
  // 停用时调用
  async deactivate() {
    console.log('插件已停用');
  }
};

export default plugin;

基本使用

安装插件

# 从 npm 安装
> /plugin install claude-code-plugin-my-plugin

# 从本地安装
> /plugin install /path/to/plugin

# 从 Git 安装
> /plugin install github:user/repo

管理插件

# 列出已安装插件
> /plugin list

# 启用插件
> /plugin enable my-plugin

# 禁用插件
> /plugin disable my-plugin

# 更新插件
> /plugin update my-plugin

# 卸载插件
> /plugin uninstall my-plugin
